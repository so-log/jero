## A. 버그 (기능 오작동)

### B1. 초대 링크 "복사" 버튼이 클립보드에 복사 안 됨
- **현상**: Step3(멤버 초대)·공유의 "복사" 눌러도 클립보드에 안 들어감.
- **원하는 것**: 클릭 시 실제 복사 + "복사됨" 피드백(토스트/체크).
- **추정 원인**: `src/lib/clipboard.ts` — `navigator.clipboard`는 보안 컨텍스트(HTTPS)+사용자 제스처 필요. 실패 폴백(`execCommand`/textarea) 없거나 async 에러 삼켜짐. 성공/실패 피드백 확인.
- **관련**: `src/lib/clipboard.ts`, `Step3Members.tsx`, `ShareOverlay.tsx`, `SharePublicBar.tsx`
- **규모**: S

### B2. 생성 시 내가 고른 나라/도시가 아니라 "도쿄"로 저장됨
- **현상**: 나라/지역을 입력/선택해도 여행이 도쿄로 저장됨.
- **원하는 것**: 선택한 나라·지역/도시가 그대로 저장·표시.
- **추정 원인**: (1) 나라/지역이 placeholder만 있고 값이 폼에 안 담김(register 누락 가능) → 빈 값이면 어딘가 "도쿄"로 폴백. (2) `create_trip` RPC/`useCreateTrip`가 country/region 무시. **F2(select화)와 함께 해결 권장.**
- **관련**: `Step1Info.tsx`, `CreateTripWizard.tsx`, `useCreateTrip.ts`, `0002_data.sql`(create_trip)
- **규모**: S~M

### B3. 여행 날짜(기간)를 생성 후 수정할 수 없음
- **현상**: 생성 뒤 시작일/종료일 변경 UI 없음.
- **원하는 것**: 여행 설정/상단에서 기간 편집 → 저장.
- **관련**: `WorkspaceTopBar.tsx`(진입점), 신규 `useUpdateTrip`, trip 설정 오버레이
- **규모**: M

### B4. "목록" 버튼이 여행 목록으로 이동 안 됨
- **현상**: 워크스페이스 상단 "← 목록" 눌러도 `/trips`로 안 넘어감.
- **원하는 것**: 클릭 시 여행 목록으로 이동.
- **추정 원인**: 버튼에 onClick/Link 미연결(죽은 버튼) 또는 라우팅 경로 오류.
- **관련**: `WorkspaceTopBar.tsx`
- **규모**: S

### B6. 플랜 Day별 "장소 추가"가 그 날짜에 배정 안 됨
- **현상**: 플랜의 "Day N 일정" 섹션에서 `+ 장소 추가`로 장소를 넣어도, 해당 Day에 안 들어가고 **장소 탭에 미배정 저장만** 됨. 헷갈림.
- **원하는 것**: Day 맥락에서 추가하면 **그 Day에 자동 배정**(scheduled_day=N, 순서 append) + 지도 동선에도 반영.
- **추정 원인**: Day별 "장소 추가" 버튼이 일반 추가 오버레이만 열고 **타깃 Day를 안 넘김** → 저장 시 미배정.
- **해결 방향**: 오버레이 프리필/컨텍스트에 targetDay 전달 → 저장 시 `useAddPlaceToSchedule`(또는 배정)로 그 Day에 넣기. *일반 "장소 추가"(장소 탭)는 미배정 유지, Day 맥락 추가만 배정.*
- **관련**: `PlanView.tsx`/`DayTimeline.tsx`(Day별 추가 버튼), `PlaceDetailOverlay.tsx`, `useAddPlaceToSchedule.ts`, `overlayStore.ts`
- **규모**: M

### B5. 플랜 ↔ 일정표 선택 날짜 미동기화
- **현상**: 플랜에서 4/11을 보다 일정표 탭으로 가면 4/11이 아닌 다른 날짜를 보여줌.
- **원하는 것**: 두 뷰가 **선택 날짜(활성 Day)를 공유** — 플랜에서 보던 날짜가 일정표에도 유지.
- **추정 원인**: `planStore`와 `calendarStore`가 선택 날짜를 각각 따로 관리 → 뷰 전환 시 동기화 안 됨.
- **해결 방향**: 선택 날짜(활성 Day)를 단일 소스로 통합하거나 두 스토어 간 동기화. *플랜/일정표 분리 자체는 의도(공간 렌즈 vs 시간 렌즈)지만 선택 상태는 공유돼야 함.*
- **관련**: `planStore.ts`, `calendarStore.ts`, `PlanView.tsx`, `CalendarView.tsx`, `ViewSegment.tsx`
- **규모**: M

## E. UX 배치·라벨

### U1. 초대 링크 공유를 멤버 목록 최상위로
- **현상**: "초대 링크로 공유"가 멤버 목록 맨 아래에 있음.
- **원하는 것**: 목록 최상위로 이동(초대가 우선 액션).
- **관련**: `Step3Members.tsx`, `ShareOverlay.tsx`
- **규모**: S

### U2. 두 개의 "전체" 혼란 — 폴더축 vs 카테고리축 IA 정리
- **현상**: 장소 뷰에 필터 축이 **둘**인데 각각 "전체"가 있어 헷갈림.
  - **폴더축**("전체 폴더"): 사용자가 만든 그룹(맛집투어·쇼핑 등) 필터.
  - **카테고리축**("전체" 칩): 장소 고정 종류(식당/카페/명소…) 필터.
  - 둘은 AND로 동시 적용됨(예: 전체 폴더 + 카페). 하지만 "전체"가 두 개 나란히 = 혼란.
- **원하는 것(택1)**:
  1. **라벨 구분(쉬움)**: 카테고리 "전체" → "모든 종류", 폴더 "전체 폴더" → "모든 폴더".
  2. **시각 분리**: 폴더 필터/카테고리 필터를 구역·구분선으로 명확히 나눔.
  3. **IA 재검토(큰 결정)**: 폴더·카테고리가 겹친다면 하나로 합치거나 한 축 제거.
- **관련**: `FolderSidebar.tsx`, `CategoryBar.tsx`, `PlacesView.tsx`(ALL_FOLDER)
- **규모**: S(라벨) ~ M(IA 재검토)

### U4. "일정표" 탭 → "캘린더"로 리네이밍
- **현상**: "플랜"과 "일정표"가 단어로 잘 구분 안 됨.
- **원하는 것**: `일정표` → **`캘린더`**. "플랜=동선/지도, 캘린더=달력/시간" 대비 명확화.
- **관련**: `ViewSegment.tsx`(탭 라벨), 관련 문구·라우팅 표기(`?view=calendar`는 유지 가능)
- **규모**: S

### U3. 카테고리 필터 칩 가로 스크롤 → 드롭다운
- **현상**: 카테고리 필터(전체·식당·카페·기념…)가 가로로 넘쳐 스크롤해야 함. 특히 모바일에서 불편.
- **원하는 것**: 가로 스크롤 칩 대신 **드롭다운(select)** 으로. 좁은 폭에서 한 번에 선택 가능.
- **고려**: 데스크톱 넓은 폭에선 칩 유지하고 좁을 때만 드롭다운으로 전환하는 반응형도 옵션. (반응형 작업과 연계 가능)
- **관련**: `CategoryBar.tsx` 또는 `PlacesView.tsx`의 카테고리 필터, `components/ui`
- **규모**: S~M
