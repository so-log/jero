# 07. 워크스페이스 — 예산 / 정산

> 여행 지출을 기록하면 카테고리별 비중·일자별 추이·1인당 금액·더치페이 정산이 자동 계산되는 대시보드. (데이터 밀집 어드민 UI 역량 — 차트·테이블·정산 알고리즘 — 의 핵심 쇼케이스)

## 1. 개요
- **화면 ID**: `workspace-budget`
- **라우트**: `/trips/[id]?view=budget`
- **목적**: 지출 내역을 기록·관리하고, 요약 지표·차트·**더치페이 정산**을 자동 산출해 누가 누구에게 얼마를 보내면 되는지까지 보여준다.

## 2. 사용자 & 진입 경로
- **사용자**: 해당 여행 멤버 (owner / editor / viewer)
- **진입**: 상단 뷰 세그먼트에서 "예산" 선택
- **이탈**: "목록" → `/trips` · 뷰 세그먼트 → 다른 뷰 · "공유" → 공유 오버레이 · "지출 추가" → 지출 추가 오버레이

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/예산 정산.dc.html`

**상단 바 (워크스페이스 공통 — 04~06과 동일)**
- "목록", 여행 아이콘+제목+기간, viewer "보기 전용" 배지
- 중앙 뷰 세그먼트(플랜/일정표/장소/**예산**), "예산" 강조
- 접속 중 멤버, `notViewer` → 공유+더보기 / `viewer` → "공유받은 플랜" 배지

**본문 (세로 스크롤)**
- **요약 지표 카드 4종**: 총지출 / 1인당 금액(N명 균등) / 남은 예산(설정 예산 대비 진행 바·N% 사용) / 최다 지출(카테고리 + 금액)
- **차트 행**:
  - 카테고리별 지출 **도넛** + 범례(카테고리·금액 비중 %)
  - 일자별 지출 추이 **막대** + 하루 평균
- **정산 + 테이블 행**:
  - **정산 요약**: "N건으로 정산" 배지 + 정산 행(보낸 사람 → 받는 사람 + 금액) + "정산 완료로 표시" 버튼(편집 권한)
  - **지출 내역 테이블**: "지출 추가" 버튼(편집 권한), 컬럼 = 날짜 / 항목(카테고리 칩+이름+카테고리 라벨) / 결제자(아바타+이름) / 분담(참여 멤버 아바타 묶음) / 금액

## 4. 데이터
- **`Expense`**(신규): id, tripId, date, name, **category**(지출 카테고리), **payerId**(결제한 멤버), **splitMembers**(분담 멤버 — 누가 N분의 1), amount, currency
- **`BudgetSetting`**: tripId, totalBudget(설정 예산 — "남은 예산"·진행 바 기준)
- **`Settlement`**(파생/계산): 지출·결제자·분담에서 산출되는 송금 목록(`from → to → amount`), 정산 완료 상태(`settledAt`)
- **파생 지표**(서버 또는 셀렉터): 총지출, 1인당(총지출/멤버수), 남은 예산(budget − 총지출), 카테고리별 합계(도넛), 일자별 합계(막대), 최다 지출 카테고리, 하루 평균
- **Category**(지출): 식비·숙소·쇼핑·교통·카페·명소·기념품 — **04~06 일정 카테고리와 일부 다름**(교통 추가, '식비'/'식당' 라벨 차이) → §13
- `Membership`(멤버 색·접속) — 결제자·분담·정산 아바타

## 5. 상태
시안 상태 스위처(`data` / `empty` / `loading` / `viewer`)에 대응.
- **내역 있음(data)**: 지표·차트·정산·테이블 전체 렌더, 편집 가능.
- **빈 상태(empty)**: 지출 0 → "아직 지출 내역이 없어요" + (편집 권한) 지출 추가 CTA.
- **로딩(loading)**: 지표·차트 스켈레톤.
- **보기 전용(viewer)**: 읽기 전용 — "지출 추가"·"정산 완료" 버튼 없음, 차트·정산은 동일 표시.
- **에러**: 조회 실패 시 재시도 안내(전역 패턴).

## 6. 인터랙션
1. (편집) "지출 추가" → 지출 추가 오버레이(오버레이 3종 #expense) → 항목·금액·카테고리·결제자·분담 입력 → 저장 시 지표·차트·정산 재계산
2. 지출 행 선택/편집·삭제(편집 권한) → 재계산
3. (편집) "정산 완료로 표시" → 정산 상태 갱신
4. 차트/지표는 데이터 변경 시 자동 갱신(읽기 전용 시각화)
5. 테이블 스크롤(많은 내역)
6. 뷰 세그먼트(플랜/일정표/장소/예산) → 같은 여행 다른 뷰로 라우팅(`?view=`)
7. (notViewer) "공유" → 공유 오버레이 · "목록" → `/trips`
8. **실시간**: 접속 멤버·동시 편집 반영(04~06과 동일)

## 7. 권한 분기
- **owner / editor**: 지출 추가·편집·삭제, 예산 설정, "정산 완료로 표시".
- **viewer**: 읽기 전용. "보기 전용"·"공유받은 플랜" 배지, 지출/정산 편집 UI 없음(차트·내역은 열람).
- 권한은 서버/RLS 강제(§8.2). 금액·정산은 서버에서 계산·검증(클라 계산 신뢰 금지, §8.3).

## 8. 라우팅 & 연결
- **들어오기**: 워크스페이스 다른 뷰의 "예산" 세그먼트
- **나가기**:
  - "목록" → `/trips` (§02)
  - 플랜 → `?view=plan` (§04) · 일정표 → `?view=calendar` (§05) · 장소 → `?view=places` (§06)
  - 공유 → 공유 오버레이 (§10) · 지출 추가 → 지출 추가 오버레이 (§10)
- 시안 내 네비게이션(참고): 세그먼트 → 각 뷰 `.dc.html`, 공유/지출추가 → `오버레이 3종.dc.html#{share|expense}`

## 9. 구현 메모
- **라우팅**: 04~06과 동일 `/trips/[id]` + `?view=budget`. **워크스페이스 공통 셸** 공유, 본문(대시보드)만 교체.
- **데이터**: `features/budget`의 TanStack Query(지출·예산 설정·정산). 지출 추가/수정 뮤테이션 → 지표·차트·정산 캐시 무효화. 컴포넌트 직접 fetch 금지(§7.1).
- **정산 계산**: **서버에서 계산**(결제자·분담 기준 채무 → 송금 건수 최소화). "N건으로 정산"은 단순화 알고리즘 결과. 클라는 표시만(§8.3).
- **차트**: Recharts(스택 §2) — 도넛(카테고리 비중), 막대(일자별 추이). 색은 카테고리 단일 출처 토큰.
- **금액 포맷**: 통화·천단위 포맷 유틸(`lib`). 통화 단일(₩) vs 다중(현지 ¥ 환산)은 §13.
- **상태 경계**: 서버상태(지출·예산·정산)=TanStack Query, UI 상태(테이블 정렬/필터 등)=로컬. 파생 지표는 가능하면 서버 응답 또는 순수 셀렉터(`lib`).
- **컴포넌트 매핑**: `features/budget`(MetricCard·CategoryDonut·DailyTrend·SettlementSummary·ExpenseTable). 공통 테이블은 `components/ui`.
- **카테고리**: 지출 카테고리 enum을 단일 출처로. 일정 카테고리(04~06)와 통합할지 분리할지 §13 → `데이터모델_계약.md`에서 확정.
- **보안**: 멤버십·역할 서버 검증(§8.2), 금액/정산 서버 계산(§8.3), 통화·숫자 입력 검증.

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거.

- [ ] `/trips/[id]?view=budget` 진입 시 지표 카드 4종 + 차트 + 정산 + 지출 테이블이 시안대로 렌더되고, 세그먼트의 "예산"이 강조된다.
- [ ] 총지출·1인당 금액·남은 예산(진행 바)·최다 지출이 지출 데이터에서 정확히 계산되어 표시된다.
- [ ] 카테고리별 도넛과 범례 비중(%)이 합계와 일치하고, 일자별 막대 추이와 하루 평균이 표시된다.
- [ ] 정산 요약이 결제자·분담을 기준으로 "보낸 사람 → 받는 사람 + 금액"으로 표시되고, 송금 건수가 최소화된다.
- [ ] 지출 테이블에 날짜·항목(카테고리·이름)·결제자·분담 멤버·금액이 표시된다.
- [ ] (편집 권한) "지출 추가" → 입력 저장 시 지표·차트·정산이 재계산된다.
- [ ] (편집 권한) "정산 완료로 표시"가 정산 상태를 갱신한다.
- [ ] 지출이 없으면 "아직 지출 내역이 없어요" + (편집 권한) CTA가 표시된다.
- [ ] 로딩 중 지표·차트 스켈레톤이 표시된다.
- [ ] viewer는 "보기 전용"/"공유받은 플랜" 배지가 보이고 지출/정산 편집 UI가 없으며, 서버에서도 편집이 거부된다(§8.2).
- [ ] 금액·정산이 서버 계산값과 일치한다(클라 단독 계산에 의존하지 않는다, §8.3).

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 지출 변경 후 지표·차트·정산 즉시 반영 / 금액·통화 포맷 일관 / 테이블·버튼 키보드 접근.

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정)
- **대시보드 구성** 고정: 지표 카드 4 + (도넛·막대) 차트 + 정산 요약 + 지출 테이블.
- **더치페이 정산** 포함: 결제자·분담 기준 자동 계산 + 송금 건수 최소화 + "정산 완료" 표시.
- **예산 설정**(totalBudget) 기반 "남은 예산"·진행 바 제공.
- **워크스페이스 공통 셸** 04~06과 공유.
- **금액·정산은 서버 계산**(클라 표시 전용).
- **viewer 읽기 전용**.

## 13. 열린 질문 (설계 단계에서 확정)
- **지출 카테고리 vs 일정 카테고리**: 통합 enum(교통 등 추가)으로 갈지, 예산 전용 카테고리를 분리할지. ('식비'/'식당' 등 라벨 정리 포함)
- **분담(split) 모델**: 균등 N분의 1만인지, 특정 멤버 지정·금액별 분담까지인지(정산 정확도·입력 UX 영향).
- **통화 처리**: 단일 통화(₩) vs 다중 통화(현지 ¥ + 환율 환산). 해외여행 특성상 중요.
- **예산 설정 위치**: totalBudget을 어디서 설정하는지(여행 생성 03엔 없음 → 예산 화면 내 설정 vs 여행 설정).
- **정산 알고리즘 상세**: 채무 단순화 규칙, 정산 완료 후 추가 지출 처리.
- **일정 항목과 지출 연계**: 04~06의 장소/일정과 지출을 연결할지(시안은 독립) — MVP 분리 권장 여부.
