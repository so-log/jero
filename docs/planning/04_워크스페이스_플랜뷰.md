# 04. 워크스페이스 — 플랜 뷰 (지도 + 일정)

> 제품의 핵심 화면. 지도에 장소를 순서대로 배치하고 동선을 그리며, 여러 명이 함께 편집한다.

## 1. 개요
- **화면 ID**: `workspace-plan`
- **라우트**: `/trips/[id]?view=plan` (워크스페이스 기본 뷰)
- **목적**: 선택한 여행의 하루 일정을 지도 + 순서 리스트로 짜고, 동선을 시각화한다.

## 2. 사용자 & 진입 경로
- **사용자**: 해당 여행의 소유자/편집자(편집), 뷰어(읽기)
- **진입**: 내 여행 목록(02)에서 여행 카드 클릭 / 여행 생성(03) 완료 직후 / 다른 워크스페이스 뷰에서 탭 전환

## 3. 구성 요소
> 기능명세서 4.4(순서 지정+동선), 4.3(지도), 4.6(필터)와 연결.

- **워크스페이스 공통 헤더**: 여행 제목·아이콘·기간, 접속 멤버(presence), 뷰 전환(플랜/일정표/장소/예산), 공유 버튼, 목록으로
- **지도 영역**:
  - 일정 등록 장소 → 순서 번호 마커 + 순서대로 잇는 동선(polyline)
  - 저장만 된 장소 → 구분되는(흐린) 마커
- **일정 패널**:
  - 선택한 날짜의 장소 카드 목록 (순서대로)
  - 드래그로 순서 변경 → 지도 동선 즉시 갱신
  - 각 카드: 순서, 카테고리 아이콘, 장소명, 시간(선택), 메모 미리보기
  - "장소 추가" 진입점
- **날짜 전환**: Day 1 / Day 2 … 또는 날짜 선택
- **필터**: "선택한 날짜 일정만 보기" 토글(켜면 저장 마커 숨김), 폴더·카테고리 필터

## 4. 데이터
- `Trip` (id, title, icon, country, region, startDate, endDate)
- `Membership` (role) — presence·권한
- `Place` (id, name, lat, lng, category, folderId, memo)
- `ItineraryItem` (id, date, order, placeId, startTime?, note?)

## 5. 상태
- **빈 상태**: 등록된 일정 없음 → "저장한 장소를 날짜에 추가해 보세요" + 장소 뷰 유도
- **로딩**: 지도/일정 스켈레톤
- **에러**: 지도 로드 실패, 데이터 로드 실패 안내
- **권한별(뷰어)**: 드래그 핸들·장소 추가·편집 숨김
- **실시간**: 다른 멤버가 편집 중인 카드 표시, 접속 멤버 아바타, 순서 변경 동기화

## 6. 인터랙션
1. 날짜 선택 → 해당 날짜 일정·동선 표시
2. "장소 추가" → 장소 상세/검색(오버레이 10) → 일정에 등록
3. 일정 카드 드래그 → `order` 갱신 → 지도 polyline 재계산
4. 카드 클릭 → 장소 상세 패널(오버레이 10)
5. 필터 토글 → 지도 마커/리스트 즉시 반영 (상태는 URL 쿼리에 반영해 공유 가능)

## 7. 권한 분기
| 동작 | 소유자 | 편집자 | 뷰어 |
|---|---|---|---|
| 보기 | O | O | O |
| 순서 변경·추가·메모 | O | O | ✕ |

## 8. 라우팅 & 연결
- **들어오기**: `/trips`(카드) · `/trips/new`(완료) · 다른 뷰 탭
- **나가기**: 뷰 전환(`?view=calendar|places|budget`), 공유→`/share/[token]` 미리보기, 목록→`/trips`
- 오버레이: 장소 상세, 멤버/공유, (장소 추가 흐름)

## 9. 구현 메모
- 지도: `@react-google-maps/api` — Marker, Polyline, 필요 시 클러스터링
- 순서 변경: dnd-kit (또는 TanStack) + 낙관적 업데이트
- 상태: 클라 UI 상태(선택 날짜·필터)는 Zustand, 서버 데이터는 TanStack Query
- 필터 상태 URL 동기화(`searchParams`)
- 실시간: 설계 단계에서 확정(Supabase Realtime / socket.io)
- 컴포넌트: `features/itinerary`(일정 패널·카드), `features/place`(마커), `features/trip`(헤더)
