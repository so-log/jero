# 04. 워크스페이스 — 플랜 뷰

> 여행의 핵심 작업 화면. 선택한 날짜의 일정(장소 순서·시간)을 좌측 리스트로, 우측 지도에 동선으로 동시에 보여주고, 여러 명이 실시간으로 함께 편집한다.

## 1. 개요
- **화면 ID**: `workspace-plan`
- **라우트**: `/trips/[id]?view=plan` (워크스페이스 **기본 뷰**)
- **목적**: 선택 날짜의 일정·동선을 리스트+지도로 동시에 보여주고, 멤버들이 실시간으로 장소 순서·구성을 함께 편집한다.

## 2. 사용자 & 진입 경로
- **사용자**: 해당 여행 멤버 (owner / editor / viewer)
- **진입**: 내 여행 목록(02) 카드 클릭 · 여행 생성(03) 완료 · 다른 뷰(일정표/장소/예산)에서 세그먼트 전환 · 공유 링크 수신(viewer)
- **이탈**: "목록" → `/trips` · 뷰 세그먼트 → 같은 여행 다른 뷰 · "공유" → 공유 오버레이 · "장소 추가" → 장소 오버레이

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 여기서는 "어떤 요소가 있고 무엇을 하는가"만 정의. 시안: `docs/design/prototype/플랜 뷰.dc.html`

**상단 바**
- "목록" 버튼(→ `/trips`)
- 여행 아이콘 + 제목("도쿄, 우리끼리 4일") + 기간("2026.4.18 – 4.21" + "3박 4일" 배지), viewer 면 **"보기 전용"** 배지
- 중앙 **뷰 세그먼트**: 플랜 / 일정표 / 장소 / 예산 (현재 뷰 강조)
- 우측: **접속 중 멤버 아바타**(온라인 점), `notViewer` → **공유** 버튼 + 더보기(⋯) / `viewer` → "공유받은 플랜" 배지

**좌측 — 일정 패널 (392px)**
- **날짜 스위처**: Day 1~N(날짜·요일), 이전/다음 화살표
- "{Day} 일정 · N곳" + **장소 추가** 버튼(편집 권한만)
- **"선택한 날짜 일정만 보기"** 토글(filterToday)
- **카테고리 바**: "전체 폴더" 드롭다운 + 전체 / 8개 카테고리 칩
- 드래그 힌트("드래그해서 순서를 바꾸면 지도 동선도 함께 업데이트돼요" — 편집+전체일 때만)
- **장소 카드 리스트**: 순번 + 카테고리 칩 + 이름 + 시간 + 카테고리 라벨 + 메모 + 드래그 핸들(편집), 선택 시 좌측 강조 바

**우측 — 지도 패널**
- 지도(실제: Google Maps), **동선 경로**(일정 순서대로 연결, 실선/점선), **번호 마커**(일정 순서), **저장 장소 마커**(다이아 — filterToday OFF일 때), **실시간 커서**(타 멤버, 이름·색)
- 범례(일정 순서 / 저장한 장소), 줌 컨트롤(+/−)

## 4. 데이터
- `Trip` (id, title, startDate, endDate) → **Day 파생**(index, date, 요일; 기간에서 계산)
- `Place` (id, tripId, **dayId**(일정 배정 시)·null(저장만), name, **category**, time?, note?, **order**, lat/lng) — **일정 장소**(날짜·순서 배정) vs **저장 장소**(미배정)
- **Category**(enum 8종): `food`(식당)·`cafe`(카페)·`gift`(기념품)·`shopping`(쇼핑)·`museum`(명소)·`hotel`(숙소)·`transport`(교통)·`etc`(기타) — 각 label·색·아이콘은 단일 출처 상수(계약 §3 단일 enum)
- `Membership` (role, 멤버 식별 색, online 상태) — 상단 아바타·커서 색에 사용
- **실시간 presence**: 접속 중 멤버, 커서(`{ memberId, color, x, y }`)
- **클라 UI 상태**: `activeDay`, `filterToday`, `activeCategory`, `selectedId` (서버 저장 안 함)

## 5. 상태
시안 상태 스위처(`editing` / `viewer` / `empty` / `loading`)에 대응.
- **편집(editing)**: 장소 추가·순서 변경(드래그)·선택 가능. 드래그는 **카테고리=전체**일 때만.
- **보기 전용(viewer)**: 읽기만. "보기 전용"/"공유받은 플랜" 배지, 추가·드래그·공유 UI 없음.
- **빈 상태(empty)**: 등록 장소 0 → 좌측 CTA("장소 추가하기") + 지도 안내 오버레이("장소를 추가하면 여기에 순서대로 동선이 그려져요").
- **로딩(loading)**: 좌측 카드 스켈레톤 + 지도 로딩 오버레이("지도를 불러오는 중…").
- **에러**: 조회 실패 시 재시도 안내(시안 외, 전역 패턴).
- **카테고리 필터 결과 0**: 리스트 비고 마커 흐림 — 빈 그룹 안내.

## 6. 인터랙션
1. 날짜 전환 → 해당 날 일정·지도 동선/번호 마커 갱신, 선택 초기화
2. "선택한 날짜만 보기" 토글 OFF → **저장 장소 마커(다이아)** 함께 표시
3. 카테고리 칩 선택 → 리스트 필터 + 비매칭 마커 흐리게(mute). **필터 중 드래그 비활성**
4. 장소 카드/지도 마커 선택 → **리스트 ↔ 지도 양방향 하이라이트**(selectedId)
5. (편집) 드래그로 순서 변경 → 리스트 순서 + 지도 동선 즉시 갱신 → 서버 반영(실시간 동기화)
6. (편집) "장소 추가" → 장소 검색/상세 오버레이(오버레이 3종 #place, §10)
7. 뷰 세그먼트(플랜/일정표/장소/예산) → 같은 여행의 다른 뷰로 라우팅(`?view=`)
8. (notViewer) "공유" → 공유 오버레이(#share)
9. "목록" → `/trips`
10. 지도 줌(+/−)·이동
11. **실시간**: 타 멤버 접속 상태·커서 표시, 동시 편집 반영

## 7. 권한 분기
- **owner / editor**: 편집(장소 추가·순서 변경·메모·삭제), 공유, 더보기.
- **viewer**: 읽기 전용. "보기 전용"·"공유받은 플랜" 배지, 편집/공유 UI 없음.
- 권한은 **서버/RLS에서 강제**(§8.2) — UI 숨김은 보안이 아니다. 실시간 편집(순서 변경 등)도 서버에서 역할 재검증.
- 공유 링크로 들어온 viewer 는 토큰 스코프(읽기 전용)로만 접근(§8.2).

## 8. 라우팅 & 연결
- **들어오기**: 02 카드 · 03 생성 완료 · 다른 뷰 세그먼트 · 공유 링크
- **나가기**:
  - "목록" → `/trips` (§02)
  - 일정표 → `/trips/[id]?view=calendar` (§05)
  - 장소 → `/trips/[id]?view=places` (§06)
  - 예산 → `/trips/[id]?view=budget` (§07)
  - 공유 → 공유 오버레이 (§10)

- 시안 내 네비게이션(참고): 세그먼트 → 각 뷰 `.dc.html`, 공유/장소추가/지출추가 → `오버레이 3종.dc.html#{share|place|expense}`

## 9. 구현 메모
- **라우팅**: `app/(app)/trips/[id]/page.tsx` + `view`는 `searchParams`(`?view=plan|calendar|places|budget`). 4개 뷰가 **상단 바·2분할 골격을 공유**하고 본문만 교체(공통 워크스페이스 레이아웃).
- **데이터**: `features/itinerary`·`features/place`·`features/member`의 TanStack Query로 trip·places·members 조회. 컴포넌트 직접 fetch 금지(§7.1).
- **실시간**: Supabase Realtime 후보(§13) — presence=접속/커서, `postgres_changes`=장소·순서 동기화. 낙관적 업데이트 + 충돌 처리(마지막 반영 기준).
- **상태 경계**: 서버상태(장소·멤버)=TanStack Query, UI 상태(`activeDay`/`filterToday`/`activeCategory`/`selectedId`)=로컬 또는 Zustand 슬라이스. 혼용 금지(§7.1).
- **드래그**: dnd 라이브러리(예: dnd-kit). 순서 변경 시 `order` 갱신 뮤테이션. **카테고리 필터 중 비활성**(시안 `canDrag` 규칙).
- **지도**: Google Maps(`@react-google-maps/api`). 번호 커스텀 마커 + Polyline 동선. 좌표는 `lat/lng`(시안 x/y%는 목업). 카테고리 mute·선택 동기화 처리.
- **카테고리**: 8종 enum을 **단일 출처 상수**(label·색·아이콘)로 — `lib/constants/category.ts` + `@theme` 토큰. 하드코딩 금지(§7.1).
- **컴포넌트 매핑**: `features/itinerary`(일정 패널·장소 카드·날짜 스위처·카테고리 바), `features/map`(지도·마커·동선·커서), 공통 세그먼트·토글·칩은 `components/ui`.
- **디자인 토큰**: 카테고리 색·멤버 식별 색·마커 스타일은 `support.js` / `디자인 시스템.dc.html`에서 추출해 `@theme` 반영(하드코딩 금지, §7.1).
- **보안**: 멤버십·역할 서버 검증(§8.2), 공유 토큰 스코프·만료(§8.2), 실시간 채널 권한 확인.

- **카테고리 8종** 고정(식당/카페/기념품/쇼핑/명소/숙소/교통/기타) — label·색·아이콘 단일 출처(계약 §3 단일 enum).
- **장소 2종**: 일정(날짜·순서·시간 배정) vs 저장(미배정). filterToday OFF에서 저장 장소 마커 노출.
- **드래그 정렬**은 편집 권한 + 카테고리=전체일 때만(시안 규칙).
- **viewer 읽기 전용**(편집·공유 불가).

## 13. 열린 질문 (설계 단계에서 확정)
- **실시간 동기화 범위**: Supabase Realtime presence/커서/장소변경 중 MVP 채택 범위(커서까지 vs 장소 동기화만). 충돌 처리 규칙.
- **장소 좌표/검색**: Google Places `place_id`·lat/lng 저장 방식, 마커 클러스터링 여부(구글맵 연동 설계와 함께).
- **시간(time) 처리**: 수동 입력 vs 미입력 허용, 시간 없는 장소의 정렬 위치.
- **저장 장소 ↔ 일정 배정 플로우**: "장소" 뷰(06)와의 관계, 드래그로 일정에 끌어다 넣는지.
- **날짜별 빈 상태**: 특정 Day만 0곳일 때 패널 표시(전체 empty와 구분).
- **동선 거리/이동 시간 표시**: 시안엔 없음 — 후속 여부.
