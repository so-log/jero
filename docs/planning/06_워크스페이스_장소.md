# 06. 워크스페이스 — 장소

> 가고 싶은 곳을 폴더에 모아두는 **저장 장소 보관함**. 여기서 날짜를 골라 "일정에 추가"하면 플랜/일정표의 일정이 된다.

## 1. 개요
- **화면 ID**: `workspace-places`
- **라우트**: `/trips/[id]?view=places`
- **목적**: 아직 일정으로 확정되지 않은 후보 장소를 폴더로 정리·검색하고, 원하는 장소를 특정 Day의 일정으로 옮긴다(플랜 04의 "저장 장소 → 일정" 배정 진입점).

## 2. 사용자 & 진입 경로
- **사용자**: 해당 여행 멤버 (owner / editor / viewer)
- **진입**: 상단 뷰 세그먼트에서 "장소" 선택
- **이탈**: "목록" → `/trips` · 뷰 세그먼트 → 다른 뷰 · "공유" → 공유 오버레이 · "장소 추가" → 장소 검색/상세 오버레이 · "일정에 추가" → 해당 Day 일정 반영(플랜/일정표에 표시)

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/장소.dc.html`

**상단 바 (워크스페이스 공통 — 04·05와 동일)**
- "목록", 여행 아이콘+제목+기간, viewer "보기 전용" 배지
- 중앙 뷰 세그먼트(플랜/일정표/**장소**/예산), "장소" 강조
- 접속 중 멤버, `notViewer` → 공유+더보기 / `viewer` → "공유받은 플랜" 배지

**좌측 — 폴더 사이드바 (236px)**
- "폴더" 헤더 + 폴더 추가(+) 버튼(편집 권한)
- 폴더 리스트: **전체 장소**(all) + 사용자 폴더(맛집·카페·쇼핑·명소·뷰·가보고 싶은 등) — 각 아이콘·이름·**개수**
- 하단 "폴더 추가" 버튼(편집 권한)

**중앙 — 장소 리스트**
- 헤더: 폴더 아이콘 + 폴더명 + "N곳" + **장소 추가** 버튼(편집 권한)
- 컨트롤: **검색**("저장한 장소 검색") + **정렬**("최근 저장순" 드롭다운)
- 상태별 본문: 로딩(스켈레톤 그리드) / 빈 상태(저장한 장소가 없어요 + CTA) / **2열 장소 카드 그리드**
- **장소 카드**: 카테고리 칩 + 이름 + 위치(area) + 카테고리 라벨 + **저장한 멤버 아바타** + 메모 + **액션**("일정에 추가" 드롭다운 / "Day N에 추가됨" + 취소 / viewer는 "저장된 장소")

**우측 — 미니 지도 (356px)**
- 현재 폴더 장소 핀 표시(선택 강조 + 이름 라벨), 일정 배정된 장소는 "D{N}" 파란 핀
- 범례("저장한 장소 · 아직 일정 아님"), 빈/로딩 상태

## 4. 데이터
- **`Folder`**(신규 — 사용자 컬렉션): id, tripId, name, icon, color. "전체 장소"는 가상 폴더(전체 집계).
- **`Place`**(04·05와 동일 엔티티, 저장 메타 관점): id, tripId, **folderId**, **category**(7종), name, **area**(위치/지역), **memo**, **savedBy**(저장한 멤버), lat/lng
  - **일정 배정 시**: `dayId`, `order`, `startTime`/`duration`, `addedBy` (04·05 필드) — 미배정이면 null = "저장만 된 장소"
- **Category** 7종(04와 동일 단일 출처)
- `Membership`(멤버 색·접속) — 상단 아바타, savedBy 아바타
- **클라 UI 상태**: 현재 `folder`, `selectedId`, 검색어, 정렬, 액션 드롭다운 열림

## 5. 상태
시안 상태 스위처(`data` / `empty` / `loading` / `viewer`)에 대응.
- **장소 있음(data)**: 폴더별 장소 카드 그리드 + 지도 핀, 편집 가능.
- **빈 상태(empty)**: 저장 장소 0 → "저장한 장소가 없어요" + (편집 권한) CTA + 지도 안내.
- **로딩(loading)**: 카드 스켈레톤 그리드 + 지도 로딩.
- **보기 전용(viewer)**: 읽기 전용 — 폴더/장소 추가·"일정에 추가" 없음, 액션 자리에 "저장된 장소" 표시.
- 검색·필터 결과 0: 해당 폴더 빈 결과 안내.

## 6. 인터랙션
1. 폴더 선택 → 해당 폴더 장소로 리스트·지도 핀 갱신, 선택 초기화
2. 장소 카드/지도 핀 선택 → **리스트 ↔ 지도 양방향 하이라이트**(이름 라벨)
3. 검색어 입력 → 폴더 내 장소 필터 / 정렬 변경 → 카드 순서 갱신
4. (편집) "장소 추가" → 장소 검색/상세 오버레이(오버레이 3종 #place)
5. (편집) "폴더 추가" → 새 사용자 폴더 생성
6. (편집) **"일정에 추가"** → 드롭다운에서 Day 선택 → 해당 일정에 배정(`dayId` 설정) → 카드에 "Day N에 추가됨" 표시, 지도 핀이 "D{N}"로 전환. "취소" → 배정 해제
7. 뷰 세그먼트(플랜/일정표/장소/예산) → 같은 여행 다른 뷰로 라우팅(`?view=`)
8. (notViewer) "공유" → 공유 오버레이 · "목록" → `/trips`
9. **실시간**: 접속 멤버·동시 편집 반영(04·05와 동일)

## 7. 권한 분기
- **owner / editor**: 장소·폴더 추가, "일정에 추가"/배정 해제, 메모 편집.
- **viewer**: 읽기 전용. "보기 전용"·"공유받은 플랜" 배지, 추가/배정 UI 없음("저장된 장소" 라벨).
- 권한은 서버/RLS 강제(§8.2). UI 숨김은 보안이 아니다.

## 8. 라우팅 & 연결
- **들어오기**: 워크스페이스 다른 뷰의 "장소" 세그먼트
- **나가기**:
  - "목록" → `/trips` (§02)
  - 플랜 → `/trips/[id]?view=plan` (§04) · 일정표 → `?view=calendar` (§05) · 예산 → `?view=budget` (§07)
  - 공유 → 공유 오버레이 (§10) · 장소 추가 → 장소 검색/상세 오버레이 (§10)
- **"일정에 추가"의 결과**는 04(플랜)·05(일정표)에 즉시 반영(같은 Place 데이터) — 화면 이동 없이 데이터 갱신
- 시안 내 네비게이션(참고): 세그먼트 → 각 뷰 `.dc.html`, 공유/장소추가 → `오버레이 3종.dc.html#{share|place}`

## 9. 구현 메모
- **라우팅**: 04·05와 동일 `/trips/[id]` + `?view=places`. **워크스페이스 공통 셸** 공유, 본문(3분할: 폴더/리스트/지도)만 교체.
- **데이터**: `features/place`의 TanStack Query(저장 장소·폴더 조회). "일정에 추가" = `dayId`/`order` 설정 뮤테이션 → 04·05 캐시 무효화로 동기화. 컴포넌트 직접 fetch 금지(§7.1).
- **저장 vs 일정 모델**: `Place`는 **folderId(저장 위치) + dayId(일정 배정, nullable)** 를 함께 가진다. 04의 "저장 장소"(미배정) = 여기서 `dayId=null`인 장소. **단일 Place 엔티티**로 통일(별도 테이블 금지) → `데이터모델_계약.md` 핵심.
- **폴더**: 사용자 생성 컬렉션(카테고리와 별개). 카테고리=고정 7종 enum, 폴더=자유 생성. 둘을 혼동하지 않게 모델 분리.
- **상태 경계**: 서버상태(장소·폴더)=TanStack Query, UI 상태(folder/selectedId/검색/정렬/드롭다운)=로컬 또는 Zustand. 폴더·검색은 URL `searchParams` 동기화 권장.
- **지도**: 04와 동일 Google Maps 컴포넌트 재사용(핀 스타일만 다이아/저장형). 일정 배정 핀은 "D{N}" 표기.
- **카테고리**: 04·05와 동일 단일 출처 상수. museum 라벨('명소' / '명소·뷰' / '명소·박물관') 시안 간 불일치 → `데이터모델_계약.md`에서 통일.
- **컴포넌트 매핑**: `features/place`(FolderSidebar·PlaceCard·PlaceGrid·일정에추가 메뉴), `features/map` 재사용. 공통 검색/정렬/버튼은 `components/ui`.
- **보안**: 멤버십·역할 서버 검증(§8.2), 장소·폴더 변경 서버 권한 확인.

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거.

- [ ] `/trips/[id]?view=places` 진입 시 폴더 사이드바 + 장소 카드 그리드 + 미니 지도가 시안대로 렌더되고, 세그먼트의 "장소"가 강조된다.
- [ ] 폴더 선택 시 해당 폴더의 장소만 리스트·지도에 표시되고, "전체 장소"는 전체를 보여준다. 각 폴더에 개수가 표시된다.
- [ ] 장소 카드 또는 지도 핀 선택 시 리스트 ↔ 지도가 양방향으로 하이라이트된다.
- [ ] 검색어로 폴더 내 장소가 필터되고, 정렬을 변경하면 카드 순서가 바뀐다.
- [ ] 장소 카드에 카테고리 칩·이름·위치·카테고리 라벨·저장한 멤버·메모가 표시된다.
- [ ] (편집 권한) "일정에 추가" → Day 선택 시 해당 장소가 그 Day 일정으로 배정되고, 카드는 "Day N에 추가됨", 지도 핀은 "D{N}"으로 바뀐다. "취소" 시 배정이 해제된다.
- [ ] "일정에 추가"의 결과가 플랜 뷰(04)·일정표(05)에 동일하게 반영된다(같은 Place 데이터).
- [ ] (편집 권한) "장소 추가"·"폴더 추가" UI가 보이고 동작한다.
- [ ] 저장 장소가 없으면 "저장한 장소가 없어요" + (편집 권한) CTA와 지도 안내가 표시된다.
- [ ] 로딩 중 카드 스켈레톤과 지도 로딩이 표시된다.
- [ ] viewer는 "보기 전용"/"공유받은 플랜" 배지가 보이고 추가·배정 UI가 없으며("저장된 장소" 표시), 서버에서도 편집이 거부된다(§8.2).

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 폴더·검색·정렬·선택 즉각 반응 / "일정에 추가" 후 다른 뷰 동기화 지연 최소 / 폴더·장소·액션을 키보드로 조작 가능.

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정)
- **3분할 레이아웃** 고정: 폴더 사이드바(236px) + 장소 리스트 + 미니 지도(356px).
- **폴더 = 사용자 생성 컬렉션**, **카테고리 = 고정 7종 enum** — 별개 개념으로 분리.
- **Place는 저장(folderId) + 일정 배정(dayId, nullable)을 한 엔티티로** 가진다 — 04의 "저장 장소"와 동일 데이터.
- **"일정에 추가" = Day 선택으로 일정 배정** 진입점(04 §13 "저장↔일정 배정 플로우" 확정).
- **워크스페이스 공통 셸** 04·05와 공유.
- **viewer 읽기 전용**.

## 13. 열린 질문 (설계 단계에서 확정)
- **폴더 범위/기본값**: 여행 단위 공유 폴더인지, 기본 폴더(맛집·가보고 싶은) 제공 vs 전부 사용자 생성. 폴더 삭제 시 장소 처리.
- **위치(area)·좌표 소스**: Google Places(`place_id`·area·lat/lng) 연동 방식 — 04·05의 좌표 결정과 함께.
- **메모 필드 통일**: 06 `memo` ↔ 04 일정 `note`가 같은 필드인지(장소 메모 vs 일정 메모 분리 여부).
- **`savedBy` vs `addedBy`**: 저장한 멤버 / 일정에 추가한 멤버를 둘 다 기록할지.
- **"일정에 추가" 시 시간 지정**: Day만 고르는지, 시각(startTime)까지 받는지(05 시간 모델과 연계).
- **중복 저장**: 같은 장소를 여러 폴더/중복 저장 허용 여부.
