# 08. 뷰어 공유 (읽기 전용)

> 공유 링크(토큰)로 들어온 비멤버가 로그인 없이 여행 일정을 **읽기 전용**으로 보는 공개 화면. 워크스페이스가 아니라 토큰 스코프의 공개 스냅샷이다.

## 1. 개요
- **화면 ID**: `share-viewer`
- **라우트**: `/share/[token]`
- **목적**: 추측 불가 토큰으로 공유된 여행의 일정·동선을 비로그인 사용자에게 읽기 전용으로 보여주고, 가입/로그인으로 유도한다.

## 2. 사용자 & 진입 경로
- **사용자**: 공유 링크를 받은 누구나(비로그인 가능). 로그인 멤버여도 이 링크로 들어오면 공개 뷰로 표시.
- **진입**: 외부에서 공유 링크 직접 진입 · 01 랜딩 "공유 링크로 둘러보기"(데모 여행, 01 §12) · 03/04의 "공유" 오버레이에서 생성된 링크
- **이탈**: "로그인하고 편집하기"/"로그인"/"무료로 시작하기" → 로그인·가입(01) · (만료 시) "홈으로"/"제이로 시작하기" → 01

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/뷰어 공유.dc.html`

**공개 상단 바 (워크스페이스 바와 다름)**
- jero 브랜드 로고 + **"읽기 전용"** 배지
- 로그인 CTA: "로그인" + "무료로 시작하기" (워크스페이스의 목록/세그먼트/공유 버튼 없음)

**상태별 본문**
- **읽기 전용 안내 배너**(normal): "읽기 전용으로 보는 중이에요. 편집하려면 로그인이 필요해요." + "로그인하고 편집하기" 버튼
- **여행 헤더**: 아이콘 + 제목 + 기간(N박N일) + **"함께하는 사람"** 멤버 아바타(읽기 전용, 관리 불가)
- **본문(일정 + 지도)** — 04 플랜 뷰의 읽기 전용 버전:
  - 좌측: 날짜 스위처 + 일정 리스트(순번·카테고리 칩·이름·시간·카테고리·메모) — **추가/드래그 없음**
  - 우측: 지도 동선(순서 연결) + 번호 마커 + 범례("{Day} 동선 · 순서대로") — **편집/줌 편집 없음**. 카드↔지도 선택 하이라이트는 가능
- **에러(만료/비활성 링크)**: "링크를 열 수 없어요" + 안내 + "홈으로"/"제이로 시작하기"
- 하단 워터마크: "제이로에서 만든 여행 일정 · 누구나 무료로 만들 수 있어요"

## 4. 데이터
- **공유 토큰**으로 조회하는 **공개 스냅샷**(인증 불필요, 읽기 전용 스코프):
  - `Trip`(title, 기간), `Day`, `Place`(일정 항목: name, category, time, note, order, 좌표) — **04와 동일 데이터의 읽기 전용 투영**
  - 멤버: 표시용 최소 정보(이니셜·식별 색)만. **이메일 등 식별정보·예산/정산은 노출 금지**(§8.5)
- **토큰 메타**: 스코프(읽기 전용), 만료/폐기 상태 → 유효성 판정(error 분기)
- 실시간 presence/커서 **없음**(공개 뷰는 정적 스냅샷)
- **클라 UI 상태**: `activeDay`, `selectedId`

## 5. 상태
시안 상태 스위처(`normal` / `loading` / `error`)에 대응.
- **정상(normal)**: 읽기 전용 안내 배너 + 여행 헤더 + 일정/지도.
- **로딩(loading)**: 일정 스켈레톤 + 지도 로딩("공유된 일정을 불러오는 중…").
- **에러(error)**: 만료·비활성·잘못된 토큰 → "링크를 열 수 없어요" + CTA. (토큰 존재 여부·여행 내용은 노출하지 않는 일반화 메시지)
- **빈 일정**(엣지): 공유된 여행에 일정이 없을 때 안내(시안 외, 04 empty 패턴 축약 적용).

## 6. 인터랙션
1. 날짜 스위처로 Day 전환 → 해당 날 일정·동선 갱신(읽기 전용)
2. 일정 카드/지도 마커 선택 → 리스트 ↔ 지도 양방향 하이라이트
3. "로그인하고 편집하기"/"로그인"/"무료로 시작하기" → 01(로그인/가입)
4. (에러) "홈으로"/"제이로 시작하기" → 01
5. 편집·추가·공유 등 **변경 동작 일절 없음**(읽기 전용)

## 7. 권한 분기
- **모두 동일한 읽기 전용**. 역할(owner/editor/viewer) 분기 없음 — 토큰 스코프 자체가 읽기 전용.
- 권한은 **서버에서 토큰으로 강제**(§8.2): 토큰이 부여한 범위(이 여행의 읽기 전용 일정)만 응답. 다른 여행·편집 API 접근 불가.
- 공개 뷰이므로 **민감 데이터(이메일·예산·정산·내부 ID) 비노출**(§8.5). UI에서 가리는 게 아니라 서버 응답에서 제외.

## 8. 라우팅 & 연결
- **들어오기**: 외부 공유 링크(`/share/[token]`) · 01 "둘러보기"(데모) · 공유 오버레이에서 생성된 링크
- **나가기**: 로그인/가입 CTA → 01(`/`, `/login`) · 만료 CTA → 01
- 시안 내 네비게이션(참고): 모든 CTA → `로그인 랜딩.dc.html`
- **연관**: 공유 링크 생성·폐기는 공유 오버레이(§10 #share)에서, 04(워크스페이스)의 viewer 모드는 이 화면의 로그인 멤버 대응

## 9. 구현 메모
- **라우팅**: `app/share/[token]/page.tsx` — 워크스페이스 레이아웃과 **별도**(공개 상단 바). 인증 미들웨어 제외(공개 경로).
- **데이터**: 토큰 기반 공개 조회 — `features/share`의 TanStack Query(또는 서버 컴포넌트). Supabase RLS/Edge로 **토큰→여행 읽기 전용 스코프** 강제. 일반 워크스페이스 쿼리(세션 기반)와 분리.
- **컴포넌트 재사용**: 04의 일정 리스트·날짜 스위처·지도(동선/마커)를 **읽기 전용 prop**으로 재사용(편집 핸들러 미주입). 중복 구현 금지 — `canEdit=false` 경로로 통일.
- **에러 처리**: 토큰 무효/만료 → error 상태. 메시지 일반화(존재 여부 비노출, §8.5).
- **보안(중요)**: 토큰 엔트로피·만료·폐기(§8.2·§8.7). 공개 응답에 민감 필드 제외(§8.5). 공유 범위 외 데이터(예산·다른 날 비공개 등) 서버에서 차단.
- **SEO/메타**: 공유 링크의 OG 메타(제목·기간) 노출 범위 → §13.
- **디자인 토큰**: 04와 동일 토큰 계승(별도 토큰 없음).

## 10. 디자인 시안 참조

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거.

- [ ] 유효한 `/share/[token]`으로 진입 시 **로그인 없이** 공개 상단 바 + 읽기 전용 안내 배너 + 여행 헤더 + 일정/지도가 시안대로 렌더된다.
- [ ] 날짜 스위처로 Day를 전환하면 해당 날 일정과 지도 동선·번호 마커가 갱신된다.
- [ ] 일정 카드/지도 마커 선택 시 리스트 ↔ 지도가 양방향 하이라이트된다.
- [ ] 추가·드래그·편집·공유 등 변경 UI가 전혀 없다(읽기 전용).
- [ ] 로그인/가입 CTA가 01(로그인/랜딩)로 이동한다.
- [ ] 만료·비활성·잘못된 토큰이면 "링크를 열 수 없어요" 에러 화면 + CTA가 표시된다(토큰/여행 존재 여부를 드러내지 않는 일반화 메시지).
- [ ] 로딩 중 일정 스켈레톤과 지도 로딩이 표시된다.
- [ ] 공개 응답에 이메일·예산/정산·내부 ID 등 민감 데이터가 포함되지 않는다(§8.5).
- [ ] 토큰이 부여한 읽기 전용 스코프 밖(편집 API·다른 여행)에는 서버에서 접근이 차단된다(§8.2).

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 비로그인 최초 로드 빠름 / 날짜·선택 키보드 조작 가능 / 공개 페이지 접근성(스크린리더 랜드마크).

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정)
- **공개 읽기 전용 뷰**: 토큰(`/share/[token]`) 접근, 로그인 불필요, 변경 동작 없음.
- **공유 본문 = 플랜(일정 + 지도)** — 04의 읽기 전용 투영.
- **공개 상단 바**(브랜드 + "읽기 전용" + 로그인 CTA), 하단 워터마크.
- **에러 상태**(만료/비활성 링크) 별도 화면 제공.
- **민감 데이터 비노출**(이메일·예산·정산) + 토큰 스코프 서버 강제.
- 01 "둘러보기"는 이 화면을 **데모 여행**으로 진입(01 §12와 연결).

## 13. 열린 질문 (설계 단계에서 확정)
- **공유 범위**: 플랜(일정)만 공유인지, 일정표/장소/예산까지 선택 공유 가능한지(MVP는 플랜만 권장).
- **토큰 수명/관리**: 만료 기간, 폐기·재발급, 링크별 권한(읽기 전용 고정 vs 추후 편집 초대 토큰 구분 — 03 초대 링크와의 관계).
- **민감 데이터 경계 확정**: 공개 응답에 포함/제외할 필드 목록(멤버 표시명 수준, 메모 노출 여부 등).
- **데모 여행 소스**: 01 "둘러보기"가 가리킬 고정 데모 여행/토큰 준비 방법.
- **OG/공유 미리보기**: 링크 공유 시 표시할 제목·썸네일 메타 범위.
- **빈 일정 처리**: 일정이 없는 여행을 공유했을 때의 표시.
