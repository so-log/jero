# 12. 초대 수락 (`/invite/[token]`)

> 편집(또는 뷰어) 초대 링크를 받은 사람이 로그인 후 여행 멤버가 되는 화면. 백엔드 연동(Supabase)으로 새로 생긴 보조 화면 — 계약 `accept_invite` RPC(데이터모델_계약 B2.1·B5)와 직접 연결된다.

## 1. 개요
- **화면 ID**: `invite-accept`
- **라우트**: `/invite/[token]` (동적 세그먼트 = `invitation.token`)
- **목적**: 추측 불가 토큰으로 받은 초대를 확인하고, 로그인/가입 후 **한 번의 액션으로 멤버 전환**(`accept_invite`)해 해당 여행 워크스페이스로 진입시킨다.

## 2. 사용자 & 진입 경로
- **진입**: 소유자(owner)가 10 멤버·공유 오버레이에서 발급한 초대 링크(`invitation` 레코드 + 링크) 클릭.
- **사용자**: 아직 이 여행의 멤버가 아닌 사람(신규/기존 계정 모두).
- **분기**: 로그인 상태 여부에 따라 (a) 바로 수락 가능, (b) 로그인/회원가입 유도 후 토큰 유지하고 복귀.
- **이탈**: 수락 성공 → 해당 여행 플랜 뷰. 에러(만료/취소/무효) → 홈 또는 로그인.

## 3. 구성 요소
> 시안: `docs/design/prototype/초대 수락.dc.html` — 로고만 있는 상단 바 + 앰비언트 배경 + 중앙 초대 카드(460px). 상태 스위처: `auth`/`guest`/`expired`/`already`.

- **초대 카드 헤더**: 초대자 아바타 + "{초대자}님이 함께하자고 초대했어요" → 여행 아이콘 타일 + 제목 + 기간 + 지역(`map-pin`).
- **역할·멤버 블록**(카드): "부여될 역할" + 역할 설명 + **역할 배지**(편집자/뷰어) · 멤버 아바타 스택 + "멤버 N명과 함께해요" + "N명 접속"(presence).
- **주 액션**: `auth`="수락하고 참여하기"(check) / `guest`="Google로 로그인하고 수락"(구글 마크) + 보조 "이메일로 로그인".
- **보조**: "나중에 할게요" · (`auth`) 하단 "{email}로 로그인됨 · 다른 계정".
- **에러 카드**(11 톤 · 중앙 일러스트): `expired`(만료/사용불가 통합 — clock·앰버) + `already`(이미 멤버 — check·그린, "여행 열기").
- **로딩**: 수락 버튼 인라인 스피너("참여하는 중…", 중복 제출 방지).

## 4. 데이터
- **초대 미리보기(읽기)**: 세션 유무와 무관하게 토큰으로 **안전 조회** — `invitation` + 대상 `trip` 요약(제목·기간·커버) + 초대자 표시명·색 + `role` + `status` + `expires_at`. **민감필드(이메일·내부 ID) 제외**(§8.5) → 별도 `security definer` 미리보기 RPC(예: `preview_invite(token)`)로 제공.
- **수락(쓰기)**: `useAcceptInvite` → `rpc('accept_invite', { token })`(B2.1). 유효성 검사(pending·미만료) → `trip_member(role=초대역할)` 전환(`on conflict do nothing`) → `invitation.status='accepted'` 소비 → `trip_id` 반환.
- 화면 자체 폼 입력 없음(로그인은 01로 위임). 토큰은 URL 세그먼트.

## 5. 상태 (시안 스위처 대응)
- **`auth`(유효·로그인)**: 초대 카드 + "수락하고 참여하기".
- **`guest`(유효·비로그인)**: 초대 카드 + 로그인 안내("참여하려면 먼저 로그인이 필요해요") + "Google로 로그인하고 수락"/"이메일로 로그인".
- **`expired`(만료·취소·무효 통합)**: 만료 에러 카드 + "내 여행 목록으로"/"홈으로". *(취소된·존재하지 않는 토큰도 존재 비노출 위해 동일 일반화 카드, §8.5)*
- **`already`(이미 멤버)**: 확인 카드 + "여행 열기".
- **수락 중**: 버튼 스피너. **성공**: `/trips/[id]?view=plan` 리다이렉트.
- **로딩**: 토큰 미리보기 조회 중(초기).

## 6. 인터랙션
1. 링크 진입 → 토큰 미리보기 조회 → (유효) 카드 렌더 / (무효·만료·취소) 에러 블록.
2. **비로그인 + 수락** → 01 로그인/회원가입으로 이동하되 `returnTo=/invite/[token]` 보존 → 인증 후 이 화면으로 복귀.
3. **로그인 + 수락** → `accept_invite` 호출 → 성공 시 `['trips']`·`['members',id]` 무효화 → `/trips/[id]` 이동.
4. **이미 멤버**(`already` · `on conflict do nothing`) → "이미 참여 중인 여행이에요" + "여행 열기".
5. **만료/취소/무효**(`expired` 통합) → 일반화 안내 + "내 여행 목록으로"/"홈으로"(존재 비노출, §8.5).

## 7. 권한 분기
- **수락은 인증 사용자만**(비로그인은 로그인 경유). 부여 역할은 **초대에 명시된 값**(editor 기본, §4.9) — 화면에서 역할 변경 불가.
- **owner는 초대로 부여 불가**(생성으로만, B2.1 `create_trip`). 초대 역할은 editor/viewer.
- 토큰·유효성·역할은 **서버(RPC)에서 강제**(§8.2). 클라는 표시·유도만.

## 8. 라우팅 & 연결
- **라우트**: `/invite/[token]` (공개 접근 가능하되 수락은 인증 필요 — 미들웨어 보호 예외 목록에 포함, B3).
- **들어오는 곳**: 외부 초대 링크(10 발급).
- **나가는 곳**: 성공 → `/trips/[id]?view=plan` · 로그인 필요 → `/`(01, `returnTo` 보존) · 에러 → `/trips` 또는 `/`.
- **연계**: 발급은 10 멤버·공유 오버레이(`useShareActions`), 수락은 본 화면(`useAcceptInvite`) — **발급/수락 분리**(B5).

## 9. 구현 메모
- `app/invite/[token]/page.tsx`(신규). 미리보기는 RSC에서 `preview_invite` 호출(서버 컴포넌트), 수락은 클라 컴포넌트 + `useAcceptInvite`(TanStack Query mutation, `features/workspace` 또는 신규 `features/invite`).
- **미인증 처리**: 미들웨어 보호 라우트에서 제외하되, "수락" 시 세션 없으면 `/?returnTo=` 로 유도(토큰 유실 방지).
- **에러 매핑**: RPC 예외(`invalid_or_expired_invite` 등) → 상태별 UI. 일반화 메시지(존재 과다 노출 금지, §8.5).
- **디자인 토큰**: 01/11 중앙 카드 패턴·역할 배지(`components/ui/role-badge`) 재사용. 하드코딩 금지(§7.1).
- **접근성**: 카드 제목 랜드마크, 액션 버튼 포커스 순서, 로딩 aria-busy.

## 10. 디자인 시안 참조
- **`docs/design/prototype/초대 수락.dc.html`** — 상태 스위처 `auth`/`guest`/`expired`/`already`. 중앙 초대 카드(460px) + 로고 상단 바 + 앰비언트 배경. DS 참조에 초대 카드 헤더·역할 배지·멤버 스택 포함.
- 연관: `로그인 랜딩.dc.html`(인증 유도·구글 버튼), `시스템 페이지.dc.html`(만료/오류 중앙 일러스트 톤).

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거.

- [ ] 유효 토큰 진입 시 여행 요약(제목·기간·초대자·부여 역할) 미리보기가 렌더된다(민감필드 미노출).
- [ ] 비로그인 상태에서 "합류"하면 로그인/회원가입으로 이동하고, 인증 후 **같은 초대 화면으로 복귀**한다(토큰 보존).
- [ ] 로그인 상태에서 수락하면 `accept_invite`로 멤버 전환되고 해당 여행 플랜 뷰로 이동한다.
- [ ] 만료·취소·무효 토큰은 **통합 만료 에러 카드**(일반화 · 존재 비노출)와 복귀 액션이 표시된다.
- [ ] 이미 멤버인 사용자는 재가입 없이 "여행으로 이동" 안내가 표시된다(중복 insert 없음).
- [ ] 부여 역할은 초대에 명시된 값이며 화면에서 변경할 수 없다(owner 부여 불가).
- [ ] 토큰·유효성·역할이 서버(RPC)에서 강제되고, 클라 조작으로 우회되지 않는다(§8.2).

**비기능**: 데스크톱 1280px 기준 + 모바일(초대는 모바일 유입 많음) 반응형 / 기존 토큰 계승 / 일반화 에러·로딩 표시 / 키보드 조작·랜드마크.

## 12. 확정 사항 (MVP 범위)
- 수락 경로 = `accept_invite` RPC(B2.1), 발급(10)과 분리. 역할은 초대 명시값(editor 기본).
- 미리보기는 민감필드 제외 안전 조회, 비로그인도 미리보기 가능(수락만 인증).
- 초대 **이메일 발송은 후속**(§알림) — MVP는 링크 공유 + 이 수락 화면.
- 토큰 만료 기본 30일(계약 B9), 재발급 가능.
