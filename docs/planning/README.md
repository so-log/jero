## 각 기획문서 공통 형식

1. **개요** — 화면 ID, 라우트, 한 줄 목적
2. **사용자 & 진입 경로** — 누가, 어디서 들어오는가
3. **구성 요소** — 화면에 들어가는 요소 (기능명세서 항목과 연결)
4. **데이터** — 사용하는 데이터 모델 / 필드
5. **상태** — 빈 / 로딩 / 에러 / 권한별 / 실시간
6. **인터랙션** — 주요 동작 흐름
7. **권한 분기** — 소유자 / 편집자 / 뷰어
8. **라우팅 & 연결** — 들어오는 곳 / 나가는 곳
9. **구현 메모** — 컴포넌트·라이브러리 매핑
10. **디자인 시안 참조** — `docs/design/prototype/`의 해당 HTML
11. **수용 기준** — 이 화면이 "done"인 조건 체크리스트 + 비기능 한 줄(반응형 기준 뷰포트·접근성·로딩). QA 테스트·GATE 승인의 근거
