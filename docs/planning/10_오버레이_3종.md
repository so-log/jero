# 10. 오버레이 3종 (장소 상세 · 멤버/공유 · 지출 추가)

> 워크스페이스 위에 뜨는 모달/패널 3종. 04~08에서 "장소 추가"·"공유"·"지출 추가"로 진입하며, 공통 모달 셸과 폼 컨트롤을 공유한다. **앞 문서들의 데이터 모델 열린 질문(역할·분담·통화·공유 권한)을 구체화하는 화면.**

## 1. 개요
- **화면 ID**: `overlay-place` / `overlay-share` / `overlay-expense`
- **표시 방식**: 라우트가 아닌 **워크스페이스 위 오버레이**(① 우측 패널, ②③ 중앙 모달). 진입은 04~08의 버튼/링크(시안 해시 `#place`/`#share`/`#expense`).
- **목적**: 장소 저장·편집, 멤버/공유 관리, 지출 입력을 흐름 끊김 없이 처리.

## 2. 사용자 & 진입 경로
- **사용자**: 해당 여행 멤버 (편집 권한 필요 동작은 owner/editor)
- **진입**:
  - ① 장소 상세 → 04 "장소 추가", 06 장소 카드/추가
  - ② 멤버·공유 → 04~07 "공유" 버튼
  - ③ 지출 추가 → 07 "지출 추가", 04~06의 "지출 추가" 동선
- **이탈**: 닫기(X)·취소·완료·저장 → 호출한 뷰로 복귀(데이터 갱신 반영)

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/오버레이 3종.dc.html`

**공통 모달 셸**: 헤더(아이콘+제목+부제+닫기) / 본문(스크롤, 에러 배너) / 푸터(액션) / 로딩 오버레이.

**① 장소 상세 · 우측 패널 (432px)**
- 미니 지도 스트립(카테고리 핀)
- 장소명(필수), 위치·주소, **카테고리 칩**(7종), **저장 폴더 선택**(맛집/카페/쇼핑/명소·뷰/가보고 싶은), 메모(textarea)
- 푸터: 삭제(휴지통) + **"일정에 추가"**

**② 멤버 · 공유 관리 · 중앙 모달 (516px)**
- **공유 링크** + 복사 + "링크로 들어온 사람의 권한"(**편집 가능 / 읽기 전용** 세그먼트 — owner만, 그 외 배지)
- **이메일 초대**(owner만): 이메일 입력 + **역할 드롭다운(기본 "편집 가능")** + 초대
- **멤버 목록**: 아바타·이름("(나)" 표시)·이메일 + **역할**(owner=배지, 그 외 변경 메뉴: 편집자/뷰어/**내보내기**)
- owner 아니면: "멤버 초대·권한 변경은 소유자만" 안내
- 푸터: 완료

**③ 지출 추가 · 중앙 모달 (480px)**
- **금액**(대형 입력 + **통화 세그먼트 KRW/JPY**) + 비-KRW 시 **기본통화 환산 안내 캡션**("1 JPY ≈ 9.2원" 류)
- 항목명, **카테고리 칩**(식비/카페/쇼핑/교통/명소/숙소)
- **결제자**(멤버 라디오 — 1명), **분담 인원**(멤버 **다중 체크** + "1인당 N원 · M명"), **날짜**(Day 칩)
- 푸터: 취소 / 저장

## 4. 데이터
- **① 장소**: 04·06의 `Place`(name·category·area/address·folderId·memo·lat/lng). "일정에 추가"=`dayId` 배정.
- **② 공유/멤버**:
  - `Membership`(memberId, **role**: owner/editor/viewer, email, 표시명·색)
  - **공유 링크/토큰**: 링크 + **권한(편집 가능 / 읽기 전용)** + 스코프(08과 연결)
  - 이메일 초대 입력(클라 일시) → 초대 생성
- **③ 지출**: 07의 `Expense`(amount, **currency: KRW/JPY**, category, **payerId**, **splitMembers**(다중), title, day). `fx_rate`는 폼 입력 아님 — 생성 시 `FX_RATES`(base=KRW) 스냅샷(데이터 계약 §7)
- **역할 정의(단일 출처)**: **소유자(owner)·편집자(editor)·뷰어(viewer)** — 03~09 전체에서 이 3종 사용
- **클라 일시 상태**: 각 폼 dirty 값, 역할 변경 메뉴 열림, 검증 표시

## 5. 상태
시안 상태 스위처(`normal` / `error` / `loading`) + ②는 **내 권한(owner / editor)** 분기.
- **normal**: 입력/표시
- **error**: 필드 검증 실패 → 상단 에러 배너 + 필드 인라인 에러 (① 장소명, ③ 금액·분담 인원, ② 초대 이메일)
- **loading**: 저장/초대 처리 오버레이("저장하는 중…"/"초대 메일을 보내는 중…")
- **권한(②)**: owner=전체 관리 / editor=링크·멤버 열람만(초대·권한변경·제거 불가, 안내 표시)

## 6. 인터랙션
**① 장소 상세**
1. 장소명·주소·카테고리·폴더·메모 편집
2. "일정에 추가" → Day 배정(또는 06 흐름) / 삭제 → 장소 제거(확인)
3. 닫기 → 06/04로 복귀, 변경 반영

**② 멤버·공유**
1. 링크 복사 / 링크 권한(편집 가능·읽기 전용) 변경(owner)
2. 이메일 초대(owner) → 초대 생성/발송(발송 범위 §13)
3. 멤버 역할 변경(편집자↔뷰어)·내보내기(owner)
4. 완료 → 닫기

**③ 지출 추가**
1. 금액·통화·항목·카테고리 입력
2. 결제자 1명 선택, 분담 멤버 다중 선택 → "1인당 N원" 실시간 계산
3. 날짜(Day) 선택
4. 저장 → 07 지표·차트·정산 재계산 / 취소 → 닫기

## 7. 권한 분기
- **① 장소 / ③ 지출**: 편집 권한(owner/editor)만 진입·저장. viewer는 진입 불가(편집 UI 자체가 없음, 08은 공개 읽기 전용).
- **② 공유**: **owner만** 초대·권한 변경·멤버 제거·링크 권한 변경. editor는 열람만.
- 모든 권한은 **서버/RLS에서 강제**(§8.2). 역할·소유 관계는 클라 입력 신뢰 금지.
- 초대/링크 권한 부여도 서버 검증(권한 상승 방지, §8.2).

## 8. 라우팅 & 연결
- 오버레이는 라우트가 아니라 **호출 뷰 위에 표시**(모달 상태 또는 `?modal=` 쿼리 — §9). 닫으면 호출 뷰 유지.
- **연결**:
  - ① ↔ 04(장소 추가)·06(장소 편집/일정 배정)
  - ② ↔ 04~07 공유 버튼·03 초대 링크·08 공유 토큰(읽기 전용 링크는 08 뷰어로 연결)
  - ③ ↔ 07 지출/지표/정산
- 시안 내 네비게이션(참고): 완료/저장/취소/일정에 추가 → `플랜 뷰.dc.html`, 공유 링크 → `뷰어 공유.dc.html`

## 9. 구현 메모
- **공통 모달 셸**: `components/ui`의 재사용 Modal/Sheet(헤더·본문·푸터·로딩·에러 배너). shadcn Dialog/Sheet 기반. 3종이 이 셸을 공유 — 중복 구현 금지.
- **표시 제어**: 워크스페이스 내 모달 상태(또는 `?modal=place|share|expense&...`로 URL 동기화 — 딥링크/뒤로가기 고려). 라우트 페이지 신설 아님.
- **폼**: React Hook Form + Zod. ① `placeSchema`, ② 초대 `inviteSchema`(이메일), ③ `expenseSchema`(금액>0·분담≥1·통화). 클라+서버 양쪽 검증(§8.3).
- **공유 컨트롤**: 카테고리 칩·아바타 선택(결제자/분담)·역할 배지/메뉴는 `components/ui` 공통 컨트롤로(04~09 재사용).
- **데이터 통신**: 저장/초대/지출 추가는 `features/*`의 뮤테이션 → 호출 뷰 캐시 무효화로 동기화. 컴포넌트 직접 fetch 금지(§7.1).
- **카테고리**: ①은 일정 카테고리, ③은 지출 카테고리 칩을 쓴다 — **두 enum의 통합/분리는 §13**(04~07과 동일 이슈).
- **보안**: 공유/역할 변경 owner 검증(§8.2), 금액·정산 서버 계산(§8.3), 초대 이메일 rate limit(§8.7).

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정 — 앞 문서 열린 질문 해소)
- **역할 3종 단일 정의**: 소유자(owner) / 편집자(editor) / 뷰어(viewer). 03~09 전체 공유.
- **공유 링크 권한 2종**: **편집 가능 / 읽기 전용** — 읽기 전용 링크는 08 뷰어로 연결(08 §13 "편집 초대 토큰" 구분 해소).
- **지출 분담 = 특정 멤버 다중 선택**(균등 N분의 1이 아니라 선택된 멤버 기준) — 결제자 1명 + 분담 멤버 다중 (07 §13 분담 모델 해소).
- **지출 통화 = KRW/JPY 2종**(지출 단위 선택) (07 §13 통화 처리 1차 해소 — 단, 09 기본 통화는 4종이라 정합 필요 §13).
- **공통 모달 셸 + 폼 컨트롤** 재사용(카테고리 칩·아바타 선택·역할 배지/메뉴).
- ① 장소는 06의 폴더·카테고리·메모 모델, "일정에 추가" 진입점 공유.
- **[B] 통화 환산** = 입력 시점 고정환율 스냅샷(`FX_RATES` base=KRW → `fx_rate` 자동, 사용자 입력 없음). 비-KRW는 환산 안내 캡션. 실시간 API phase 2.
- **[C] 초대 역할** = 기본 `editor`(드롭다운, owner 부여 불가), 초대 후 변경은 멤버 목록 드롭다운. 서버 재검증.
- **[D] 삭제 확인** = `components/ui/ConfirmDialog` 공유 컴포넌트(장소·지출·멤버·계정 재사용).

## 13. 열린 질문 (설계 단계에서 확정)
- **카테고리 enum 표시 집합**: 단일 `category` enum(데이터 계약 §3)에서 ①(일정)·③(지출) 화면별 노출 부분집합 분기 — 부분집합 정의 위치(lib).
- **공유 링크 토큰 관리**: 편집 가능 링크의 보안 경계(읽기 전용보다 위험), 만료·폐기·재발급(08·03과 통합).
- **오버레이 URL 동기화**: 모달 상태를 `?modal=`로 딥링크할지(뒤로가기·새로고침 동작).
- **초대 발송 트리거**: 메일 발송(후속) 시점·미가입자 초대 처리.
