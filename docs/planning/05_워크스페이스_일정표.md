# 05. 워크스페이스 — 일정표

> 같은 여행 데이터를 시간 축으로 보는 뷰. 월/주/일 3가지 모드로 일정을 보고, 시간·소요 시간 기준으로 하루를 구성한다.

## 1. 개요
- **화면 ID**: `workspace-calendar`
- **라우트**: `/trips/[id]?view=calendar`
- **목적**: 플랜 뷰(04)와 같은 일정 데이터를 **달력/타임라인 형태**로 보여주고, 월·주·일 단위로 일정을 확인·편집한다.

## 2. 사용자 & 진입 경로
- **사용자**: 해당 여행 멤버 (owner / editor / viewer)
- **진입**: 상단 뷰 세그먼트에서 "일정표" 선택(주로 플랜 뷰 04에서 전환)
- **이탈**: "목록" → `/trips` · 뷰 세그먼트 → 다른 뷰 · "공유" → 공유 오버레이 · "일정 추가" → 일정/장소 추가 오버레이

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/일정표.dc.html`

**상단 바 (워크스페이스 공통 — 04와 동일)**
- "목록" 버튼, 여행 아이콘+제목+기간, viewer "보기 전용" 배지
- 중앙 뷰 세그먼트(플랜/**일정표**/장소/예산), "일정표" 강조
- 접속 중 멤버 아바타, `notViewer` → 공유 버튼+더보기 / `viewer` → "공유받은 플랜" 배지

**뷰 툴바 (일정표 전용)**
- 좌측: **이전 / 오늘 / 다음** 네비게이션 + **기간 라벨**(월: "2026년 4월" / 주: "4월 18일 – 24일" / 일: "4월 18일 토요일")
- 우측: **모드 세그먼트(월 / 주 / 일)** + **일정 추가** 버튼(편집 권한만)

**본문 — 3개 모드**
- **월(month)**: 달력 그리드. 날짜 셀에 **카테고리 색 도트** + "N개 일정" 라벨, 여행 기간일 강조, 선택일 강조 원. 여행 기간일(18–21) 클릭 시 **일(day) 모드로 전환**.
- **주(week)**: 7일 타임라인(08:00–20:00), 시간 거터 + **시작 시각·소요 시간 기준 이벤트 블록**(카테고리 색 좌측 보더). 여행 기간 외 날짜는 흐림.
- **일(day)**: 세로 타임라인. 시간 거터(시작–종료) + 커넥터 도트/라인 + **일정 카드**(카테고리 칩·이름·카테고리 라벨·메모 + "추가" 한 멤버 아바타).

## 4. 데이터
> **플랜 뷰(04)와 동일한 일정/장소 데이터를 시간 축으로 재구성.** 새 필드만 추가.
- `Place`/일정 항목 (04 정의 계승: id, dayId, name, category, note, order) + **시간 관련**:
  - `startTime`(시각), **`duration`/`endTime`**(소요 시간 — 주/일 모드 블록 높이)
  - **`addedBy`**(추가한 멤버 — 일 모드 카드의 "추가 · {아바타}")
- `Trip`(기간) → Day·달력 셀 파생, 여행 기간 범위 강조에 사용
- **Category** 7종(04와 동일 단일 출처)
- `Membership`(멤버 색·접속 상태) — 상단 아바타, addedBy 아바타
- **클라 UI 상태**: `mode`(month/week/day), `selDay`(선택 날짜), 현재 기간 커서

## 5. 상태
시안 상태 스위처(`data` / `empty` / `loading` / `viewer`)에 대응.
- **일정 있음(data)**: 선택 모드(월/주/일)로 일정 렌더, 편집 가능.
- **빈 날(empty)**: 선택 날짜에 일정 0 → "이 날은 아직 일정이 없어요" + 일정 추가하기 CTA(편집 권한만).
- **로딩(loading)**: 달력 스켈레톤.
- **보기 전용(viewer)**: 모드별 렌더는 동일하나 읽기 전용("일정 추가" 없음, 편집 불가).
- **에러**: 조회 실패 시 재시도 안내(전역 패턴).

## 6. 인터랙션
1. 모드 세그먼트(월/주/일) 전환 → 본문·기간 라벨 갱신
2. 이전/다음 → 현재 모드 단위로 기간 이동, "오늘" → 기준일로 복귀(여행 첫날)
3. (월) 날짜 셀 클릭 → 선택일 갱신, 여행 기간일이면 일(day) 모드로 진입
4. 이벤트 블록/카드 클릭 → 일정 상세/편집(오버레이 — §13)
5. (편집) "일정 추가" → 일정/장소 추가 오버레이
6. 뷰 세그먼트(플랜/일정표/장소/예산) → 같은 여행 다른 뷰로 라우팅(`?view=`)
7. (notViewer) "공유" → 공유 오버레이 · "목록" → `/trips`
8. **실시간**: 접속 멤버·동시 편집 반영(04와 동일 협업 기반)

## 7. 권한 분기
- **owner / editor**: 일정 추가·편집, "일정 추가" 버튼 노출.
- **viewer**: 읽기 전용. "보기 전용"·"공유받은 플랜" 배지, 추가/편집 UI 없음.
- 권한은 서버/RLS 강제(§8.2). UI 숨김은 보안이 아니다.

## 8. 라우팅 & 연결
- **들어오기**: 워크스페이스 다른 뷰의 "일정표" 세그먼트
- **나가기**:
  - "목록" → `/trips` (§02)
  - 플랜 → `/trips/[id]?view=plan` (§04)
  - 장소 → `/trips/[id]?view=places` (§06)
  - 예산 → `/trips/[id]?view=budget` (§07)
  - 공유 → 공유 오버레이 (§10) · 일정 추가 → 추가 오버레이 (§10)
- 시안 내 네비게이션(참고): 세그먼트 → 각 뷰 `.dc.html`, 공유 → `오버레이 3종.dc.html#share`

## 9. 구현 메모
- **라우팅**: 04와 동일하게 `/trips/[id]` + `?view=calendar`. **워크스페이스 공통 레이아웃**(상단 바·뷰 세그먼트)을 04와 공유하고 본문만 교체. 뷰 툴바(이전/오늘/다음·모드 세그먼트)는 일정표 전용.
- **데이터**: 04와 **동일한 일정 데이터 소스**를 시간 축으로 변환. 별도 fetch 없이 `features/itinerary`의 TanStack Query 재사용. 컴포넌트 직접 fetch 금지(§7.1).
- **모드 상태**: `mode`(month/week/day)·`selDay`는 UI 상태(URL `searchParams` 동기화 권장 — 새로고침/공유 복원). 서버상태와 분리(§7.1).
- **달력 계산**: 월 그리드(앞뒤 달 채움), 주 7일, 시각→블록 위치/높이 매핑. **타임존 주의**(여행 현지 기준 정의 필요 — §13).
- **컴포넌트 매핑**: `features/itinerary`(MonthGrid·WeekTimeline·DayTimeline·뷰 툴바). 공통 세그먼트·버튼은 `components/ui`. 04의 카테고리 칩·멤버 아바타 재사용.
- **카테고리**: 04와 **동일 단일 출처 상수**(label·색·아이콘). 시안의 museum 라벨('명소')은 04('명소·박물관')와 통일 필요 → `데이터모델_계약.md`에서 확정.
- **디자인 토큰**: 달력 셀·이벤트 블록·타임라인 카드 스타일은 시안/디자인 시스템에서 추출 `@theme`(하드코딩 금지, §7.1).
- **보안**: 멤버십·역할 서버 검증(§8.2).

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거.

- [ ] `/trips/[id]?view=calendar` 진입 시 워크스페이스 상단 바 + 일정표 본문이 시안대로 렌더되고, 세그먼트의 "일정표"가 강조된다.
- [ ] 모드 세그먼트(월/주/일) 전환 시 본문과 기간 라벨이 해당 모드로 바뀐다.
- [ ] 이전/다음/오늘 네비게이션이 현재 모드 단위로 기간을 이동·복귀시킨다.
- [ ] 월 모드에서 날짜 셀에 카테고리 도트와 "N개 일정"이 표시되고, 여행 기간일 클릭 시 일 모드로 전환된다.
- [ ] 주 모드에서 이벤트가 시작 시각·소요 시간에 맞는 위치·높이의 블록으로 표시된다.
- [ ] 일 모드에서 시간 순 타임라인 카드(카테고리·이름·메모·추가한 멤버)가 표시된다.
- [ ] 선택 날짜에 일정이 없으면 "이 날은 아직 일정이 없어요" + (편집 권한) 추가 CTA가 표시된다.
- [ ] 로딩 중 달력 스켈레톤이 표시된다.
- [ ] (편집 권한) "일정 추가" 버튼이 보이고 추가 오버레이가 열린다.
- [ ] viewer는 "보기 전용"/"공유받은 플랜" 배지가 보이고 추가/편집 UI가 없으며, 서버에서도 편집이 거부된다(§8.2).
- [ ] 일정표에서 본 데이터가 플랜 뷰(04)와 동일한 출처를 사용한다(불일치 없음).

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 모드·기간 전환 즉각 반응 / 시각·소요 시간 매핑이 타임존 기준으로 정확 / 모드·날짜를 키보드로 조작 가능.

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정)
- **3개 모드** 고정: 월 / 주 / 일.
- **워크스페이스 공통 셸** 04와 공유(상단 바·뷰 세그먼트·실시간 협업 기반).
- **일정 데이터는 04와 단일 출처** — 일정표는 시간 축 표현일 뿐 별도 데이터 아님.
- **카테고리 7종** 04와 동일(label·색·아이콘 단일 출처).
- 시각 정보 모델에 **`startTime` + `duration`(소요 시간)** 포함, 일정 항목에 **추가한 멤버(`addedBy`)** 기록.

## 13. 열린 질문 (설계 단계에서 확정)
- **일정 추가/편집 플로우**: "일정 추가" 및 이벤트 클릭이 04의 장소 추가 오버레이를 공유하는지, 일정표 전용 시간 편집 UI가 있는지.
- **시간/소요 시간 입력**: `duration` 수동 입력 vs 종료 시각 입력, 시간 미지정 일정의 표시 위치(주/일 모드).
- **타임존 기준**: 여행 현지 시간 vs 사용자 로컬 — 달력·블록 계산의 기준 시간대.
- **드래그 이동**: 주/일 모드에서 블록을 드래그해 시간 변경 지원 여부(MVP 포함 여부).
- **`addedBy` 의미**: "작성자" 표시인지 "담당자" 지정인지(권한·필터 영향).
- **기간 밖 날짜 편집**: 여행 기간(18–21) 외 날짜에 일정 추가 허용 여부.
