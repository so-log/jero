# 09. 계정 설정

> 내 프로필(다른 멤버에게 보이는 정보)·기본 설정(통화·알림)·계정 관리(로그아웃·삭제)를 한 화면에서 관리한다.

## 1. 개요
- **화면 ID**: `settings`
- **라우트**: `/settings`
- **목적**: 로그인 사용자가 프로필·환경설정을 변경하고 계정을 관리(로그아웃·삭제)한다.

## 2. 사용자 & 진입 경로
- **사용자**: 로그인 사용자(본인 계정만)
- **진입**: 02/워크스페이스의 사용자 메뉴 → "계정 설정"
- **이탈**: "목록" → `/trips` · 로그아웃 → 01 · 계정 삭제 완료 → 01

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/계정 설정.dc.html`

**상단 바**
- "목록" 버튼 + "계정 설정" 타이틀 + (우측) 현재 사용자 아바타 + 이름/이메일

**좌측 섹션 내비게이션 (230px)**
- 프로필 / 기본 설정 / 계정 관리 (현재 섹션 강조)
- 하단: 버전 + 마지막 로그인 정보

**본문 (섹션별)**
- **프로필** ("다른 멤버에게 보이는 정보"):
  - 프로필 사진: 아바타 + 업로드/삭제 + **아바타 색상 피커(5색)**
  - 이름(편집, "멤버 목록과 커서에 표시")
  - 이메일(읽기 전용, lock + "인증됨" 배지, 로그인 이메일)
- **기본 설정** ("통화와 알림"):
  - **기본 통화** 세그먼트(원 ₩ / 엔 ¥ / 달러 $ / 유로 €) — "예산·지출에 기본으로 쓰여요"
  - **알림 토글 4종**: 여행 업데이트 / 댓글·멘션 / 정산 알림 / 소식·마케팅
- **계정 관리**:
  - 로그아웃("이 기기에서 로그아웃")
  - **계정 삭제**(위험 액션) → 확인 모달("정말 계정을 삭제할까요?" + 영구 삭제 경고 + 취소/삭제할게요)

**하단 저장 바**
- 저장 상태 안내(idle/saving/saved/error) + "되돌리기" + **"변경사항 저장"**(명시적 저장)

## 4. 데이터
- `User` 프로필: name(편집), email(읽기 전용·인증 상태), avatar(**color** + 이미지)
- `UserPreference`: defaultCurrency(KRW/JPY/USD/EUR), notif{ trip, comment, settle, marketing }(불리언)
- 메타: 앱 버전, 마지막 로그인 시각
- **아바타 색상은 멤버 식별 색**(04~08의 멤버 색·커서·아바타와 동일 소스)
- **클라 일시 상태**: 폼 dirty 값, 현재 섹션, 저장 상태, 삭제 모달 열림

## 5. 상태
시안 상태 스위처(`idle` / `saving` / `saved` / `error`) = **저장 플로우 상태**.
- **기본(idle)**: "변경사항은 저장을 눌러야 적용돼요"
- **저장 중(saving)**: 스피너 + "저장하는 중…", 저장 버튼 비활성
- **저장 완료(saved)**: "모든 변경사항이 저장되었어요"
- **오류(error)**: "저장에 실패했어요. 다시 시도해 주세요."
- **삭제 확인 모달**: 위험 액션 별도 확인(취소/삭제)
- (로딩/초기 조회는 전역 패턴)

## 6. 인터랙션
1. 좌측 섹션 이동(프로필/기본설정/계정관리) → 해당 섹션 표시
2. 이름·아바타 색·사진·통화·알림 변경 → 폼 dirty, **저장 눌러야 반영**(되돌리기로 취소)
3. "변경사항 저장" → saving → saved/error
4. 프로필 사진 업로드/삭제
5. 로그아웃 → 세션 종료 → 01
6. 계정 삭제 → 확인 모달 → "삭제할게요" → 계정·데이터 영구 삭제 → 01
7. "목록" → `/trips`

## 7. 권한 분기
- 본인 계정만 접근·수정(인증 필수, 보호 라우트). 타인 계정 접근 불가(§8.2).
- 이메일은 읽기 전용(인증된 로그인 이메일). 변경 가능 여부는 §13.
- 계정 삭제는 **본인 재확인 + 서버에서 인증·소유 검증**(§8.2). 파괴적 작업이므로 확인 모달 필수.

## 8. 라우팅 & 연결
- **들어오기**: 사용자 메뉴(02·워크스페이스) → `/settings`
- **나가기**: "목록" → `/trips`(§02) · 로그아웃/계정 삭제 → 01(`/`)
- 시안 내 네비게이션(참고): "목록" → `내 여행 목록.dc.html`, 로그아웃 → `로그인 랜딩.dc.html`

## 9. 구현 메모
- **라우팅**: `app/(app)/settings/page.tsx`. 보호 라우트(미인증 → 01 리다이렉트, 서버 세션 검증 §8.2). 섹션은 단일 페이지 + 좌측 nav(라우트 분리 vs 스크롤/탭은 §13).
- **데이터**: `features/account`(또는 `user`)의 TanStack Query(프로필·환경설정 조회), 저장은 뮤테이션. 컴포넌트 직접 fetch 금지(§7.1).
- **저장 모델**: **명시적 저장**(dirty → 저장 버튼). React Hook Form + Zod로 폼·검증, "되돌리기"=reset. 낙관/비낙관 정책 §13.
- **아바타 색**: 04~08의 멤버 식별 색과 **동일 소스**(User.avatarColor). 토큰 팔레트(5색)로 제한.
- **통화**: `defaultCurrency`는 07 예산의 기본 통화로 연결. 다중 통화/환율(07 §13)과 일관되게 설계.
- **알림**: 토글은 `UserPreference.notif`에 저장. **실제 발송 동작 범위는 §13**(02 §12에서 알림 MVP 제외와 충돌 — 아래 주의).
- **계정 삭제**: 파괴적·되돌릴 수 없음. cascade 정책(owner인 공유 여행 처리) §13. 서버에서 처리.
- **컴포넌트 매핑**: `features/account`(SettingsNav·ProfileSection·PreferenceSection·DangerZone·SaveBar). **계정 삭제 확인은 `components/ui/ConfirmDialog`(공유 컴포넌트)를 사용**(장소·지출·멤버 삭제와 공용, account 전용 모달 두지 않음). 공통 토글·필드·세그먼트·모달은 `components/ui`.
- **보안**: 보호 라우트·본인 검증(§8.2), 이메일/비밀번호 변경 시 재인증 고려(§8), 입력 검증(§8.3).

## 11. 수용 기준 (Acceptance Criteria)

- [ ] `/settings` 진입 시 상단 바 + 좌측 섹션 nav + 프로필/기본설정/계정관리 본문 + 하단 저장 바가 시안대로 렌더된다.
- [ ] 좌측 nav로 섹션을 전환할 수 있고 현재 섹션이 강조된다.
- [ ] 이름·아바타 색·사진·통화·알림을 변경하면 "변경사항 저장"을 눌러야 반영되고, "되돌리기"로 변경이 취소된다.
- [ ] 저장 시 saving → saved 상태가, 실패 시 error 상태가 표시된다.
- [ ] 이메일은 읽기 전용으로 "인증됨" 상태와 함께 표시된다.
- [ ] 아바타 색상 변경이 멤버 표시 색(04~08 커서·아바타)과 일치한다.
- [ ] 기본 통화 선택이 저장되고 예산(07)의 기본 통화로 사용된다.
- [ ] 로그아웃 시 세션이 종료되고 01로 이동한다.
- [ ] 계정 삭제는 확인 모달을 거쳐서만 실행되며, 완료 시 계정·데이터가 삭제되고 01로 이동한다.
- [ ] 미인증 상태로 `/settings` 접근 시 01로 리다이렉트되고, 타인 계정 데이터에 접근할 수 없다(§8.2).

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 저장 상태 피드백 명확 / 파괴적 액션(삭제) 확인 필수 / 폼·토글·모달 키보드 접근.

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정)
- **3섹션 구성**(프로필 / 기본 설정 / 계정 관리) + 좌측 nav + 하단 저장 바.
- **명시적 저장**(되돌리기 + 변경사항 저장) + 저장 상태(idle/saving/saved/error).
- **프로필**: 이름(편집)·이메일(읽기 전용)·아바타(색상 + 사진).
- **아바타 색상 = 멤버 식별 색**(04~08 단일 소스).
- **기본 통화 설정** 제공(예산 기본 통화로 연결).
- **계정 삭제 = 확인 모달 + 영구 삭제**(서버 처리).

## 13. 열린 질문 (설계 단계에서 확정)
- **알림 동작 범위(중요)**: 토글 UI는 고정 시안에 있으나 02 §12에서 알림 MVP 제외와 충돌. → 설정 저장만(실제 발송 없음) vs 실제 푸시/이메일 발송 포함을 결정. (실시간 협업 범위·04와 함께 검토)
- **통화 정책**: 유저 기본 통화 + 07 다중 통화/환율 환산 통합 범위.
- **아바타 사진 업로드**: 색상만 MVP vs 이미지 업로드 포함(스토리지 필요).
- **이메일 처리**: "인증됨" 배지 의미(OAuth/확인 메일), 이메일·비밀번호 변경 지원 여부.
- **계정 삭제 cascade**: 본인이 owner인 공유 여행(다른 멤버 존재) 처리 — 소유권 이전 vs 여행 삭제 정책.
- **섹션 라우팅**: 단일 페이지 스크롤/탭 vs `/settings/{profile|preferences|account}` 분리.
