# 03. 여행 생성 (다단계 마법사)

> 새 여행의 골격(정보·기간·멤버·시작 방식)을 4단계로 입력해 워크스페이스를 만든다.

## 1. 개요
- **화면 ID**: `trip-create`
- **라우트**: `/trips/new`
- **목적**: 4단계 폼으로 여행 기본 정보·기간·멤버·시작 방식을 받아 Trip을 생성하고 워크스페이스로 진입시킨다.

## 2. 사용자 & 진입 경로
- **사용자**: 로그인 사용자 (생성자 = 소유자 owner)
- **진입**: 내 여행 목록(02)의 "새 여행 만들기" / 빈 상태 "첫 여행 만들기"
- **이탈**: 생성 완료 → 워크스페이스(`/trips/[id]`, 플랜 뷰) · 닫기(X) → `/trips`

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/새 여행 만들기.dc.html`

**프레임 (전 단계 공통)**
- 헤더: 로고 + "새 여행 만들기" 타이틀 + 닫기(X)
- **스테퍼**(4단계): 여행 정보 / 여행 기간 / 멤버 초대 / 시작 방식 — 완료 단계는 체크, 현재 단계 강조
- 본문(단계별 콘텐츠), 에러 배너(검증 실패 시), 생성 중 오버레이
- 푸터: 이전 / (단계 N / 4) / 다음 ↔ "여행 만들기"(마지막)

**Step 1 — 여행 정보**
- **라이브 프리뷰 배너**(커버색 + 아이콘 + 제목 + 나라·지역 실시간 반영)
- 여행 제목(필수), 대표 아이콘(6종: building/palmtree/mountain/utensils/landmark/waves), 커버 색(5종: blue/mint/coral/purple/amber), 나라, 지역/도시

**Step 2 — 여행 기간**
- 시작일 / 종료일 필드, 월 캘린더에서 **범위 선택**(시작 클릭 → 종료 클릭), 선택 시 "N박 N일" 배지

**Step 3 — 멤버 초대** (선택)
- 이메일로 초대(입력 + 추가, Enter 지원) → 멤버 목록에 추가(기본 역할: 편집 가능)
- 멤버 목록: 아바타·이름·이메일 + **역할 토글**(편집 가능 / 읽기 전용), 소유자(나)는 고정·제거 불가, 그 외 제거 가능
- **초대 링크로 공유**: 읽기 전용 스코프 링크 + 복사

**Step 4 — 시작 방식**
- 선택 카드 2종: **빈 여행으로 시작** / **템플릿 복제로 시작**
- 템플릿 선택 시 템플릿 목록 노출(라디오 선택, 미선택 시 에러)

## 4. 데이터
- 생성 입력(`createTripInput`):
  - `Trip`: title, coverIcon, coverColor, country, region, startDate, endDate
  - 멤버 초대: `[{ email, role: editor|viewer }]` (생성자는 서버가 owner로 자동 추가)
  - 시작 방식: `startMode: blank | template`, `templateId?`
- 생성 결과: `Trip.id` (→ 워크스페이스 라우트)
- 초대 링크: 읽기 전용 토큰(추측 불가, 만료/폐기 가능) — §8.2
- **클라 일시 상태**: 4단계 폼 값 + 현재 step + 검증 표시 여부 (제출 전까지 클라 보관)

## 5. 상태
시안 상태 스위처(`form` / `error` / `loading`)에 대응.
- **진행 중(form)**: 단계 이동·입력
- **검증 에러(error)**: "다음" 시 현재 단계 검증 실패 → 에러 배너 + 필드 인라인 에러
  - Step1: 제목 미입력 → "여행 제목을 입력해 주세요"
  - Step2: 기간 미선택 → "시작일과 종료일을 선택해 주세요"
  - Step4: 템플릿 모드인데 미선택 → "복제할 템플릿을 선택해 주세요"
  - Step3: 검증 없음(선택 단계)
- **생성 중(loading)**: 오버레이("여행을 만드는 중…") → 성공 시 워크스페이스
- **생성 실패(서버)**: 에러 안내 + 재시도(입력값 유지)

## 6. 인터랙션
1. 다음 → 현재 단계 클라 검증 → 통과 시 다음 단계, 실패 시 에러 상태
2. 이전 → 이전 단계로(에러 초기화), 입력값 유지
3. Step1: 아이콘/커버 선택 → 프리뷰 즉시 갱신
4. Step2: 날짜 두 번 클릭으로 범위 지정(종료 < 시작이면 시작 재설정), N박N일 계산
5. Step3: 이메일 추가/제거, 역할 토글, 초대 링크 복사
6. Step4: 시작 방식 선택, 템플릿 선택
7. 마지막 "여행 만들기" → 전체 검증 → 서버 생성 → `/trips/[id]`
8. 닫기(X) → `/trips` (입력 폐기 — 확인 모달 여부는 §13)

## 7. 권한 분기
- 생성자는 항상 **소유자(owner)**. 초대 멤버는 editor/viewer 중 지정(소유자는 생성 시 1인).
- 멤버 역할·초대는 **서버에서 검증·생성**(클라가 보낸 role/소유 관계를 신뢰하지 않음, §8.2).
- 초대 링크는 **읽기 전용 권한으로 스코프**, 추측 불가 토큰, 만료/폐기 가능(§8.2).

## 8. 라우팅 & 연결
- **들어오기**: `/trips`(새 여행 / 첫 여행 만들기)
- **나가기**:
  - 생성 완료 → `/trips/[id]` (워크스페이스 플랜 뷰, §04)
  - 닫기 → `/trips`
- 시안 내 네비게이션(참고): 생성/완료 → `플랜 뷰.dc.html`, 닫기 → `내 여행 목록.dc.html`

## 9. 구현 메모
- 폼: React Hook Form + Zod 다단계. `createTripSchema`(단계별 partial 검증 + 최종 전체 검증). 시안 검증 규칙을 스키마에 반영.
- 단계 상태: 폼 값·step은 컴포넌트/`features/trip` 로컬 상태(전역 불필요). 제출은 `features/trip/api` 뮤테이션(`useCreateTrip`).
- 캘린더: 시안은 2026년 4월 단일 월 고정 → 실제는 **월 이동 + 임의 연·월** 지원 필요(§13). shadcn Calendar/range 검토, 타임존 주의.
- 멤버 입력: 이메일 형식 검증, 중복 방지. role 기본 editor. 실제 사용자 존재 여부·초대 발송은 서버.
- 보안: 생성·초대는 서버에서 소유/권한 재확인(§8.2), 초대 링크 토큰 엔트로피·만료(§8.2·§8.7), 입력 Zod 서버 재검증(§8.3).
- 컴포넌트 매핑: `features/trip/components`(`CreateTripWizard`, 단계별 Step 컴포넌트), 공통 입력·스테퍼·선택카드는 `components/ui`.
- 디자인 토큰(아이콘/커버 세트·스테퍼 색)은 `support.js`에서 추출해 `@theme` 반영(하드코딩 금지, §7.1).

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거.

- [ ] 4단계 스테퍼가 진행에 따라 완료(체크)/현재(강조)/예정 상태를 시안대로 표시한다.
- [ ] Step1 제목 미입력, Step2 기간 미선택, Step4 템플릿 모드 미선택 시 "다음"이 차단되고 에러 배너 + 필드 인라인 에러가 표시된다.
- [ ] Step1에서 제목·아이콘·커버·나라·지역 변경이 라이브 프리뷰에 즉시 반영된다.
- [ ] Step2에서 캘린더 범위 선택이 동작하고(종료<시작이면 시작 재설정), "N박 N일"이 정확히 계산된다.
- [ ] Step3에서 이메일로 멤버를 추가/제거하고 역할(편집 가능/읽기 전용)을 토글할 수 있으며, 소유자(나)는 제거되지 않는다.
- [ ] 초대 링크는 읽기 전용 스코프로 표시되고 복사할 수 있다.
- [ ] Step4에서 빈 여행/템플릿 복제를 선택할 수 있고, 템플릿 모드에서는 템플릿 선택이 필수다.
- [ ] 마지막 "여행 만들기" 시 생성 중 오버레이가 표시되고, 성공 시 해당 여행 워크스페이스(`/trips/[id]`)로 이동한다.
- [ ] 생성자는 서버에서 소유자(owner)로 설정되고, 멤버 역할/소유 관계는 클라 입력을 신뢰하지 않고 서버에서 강제된다(§8.2).
- [ ] 생성 실패 시 입력값을 유지한 채 재시도 가능한 에러가 표시된다.

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 폼 전체 키보드 조작 가능 / 단계 이동 시 입력값 보존 / 날짜 계산 타임존 정확.

## 12. 확정 사항 (MVP 범위 · 디자인·스펙 고정)
- **단계 구성**: 4단계(여행 정보 → 기간 → 멤버 초대 → 시작 방식) 고정. 멤버 초대(Step3)는 선택(스킵 가능).
- **아이콘/커버**: 아이콘 6종 + 커버 색 5종 고정 세트(사용자 업로드 이미지 미지원 — 02 §12와 일치, `Trip`은 `coverColor`/`coverIcon`만).
- **초대 링크 권한**: 읽기 전용으로 고정.
- **시작 방식**: 빈 여행 / 템플릿 복제 2종.
- **템플릿 복제**: ✅ MVP 포함 — **정적 시드 템플릿 3종 복제**. 템플릿 생성·관리 UI 없음(하드코딩 시드를 새 Trip으로 복제만). 시안의 활성 상태와 일치.
- **멤버 초대 발송**: ✅ **초대 링크 복사 중심**. 실제 이메일 발송은 MVP 제외(02 §12 알림 제외와 일관). Step3 이메일 입력은 멤버 사전 등록용이며, 실제 합류는 링크 공유로 처리.

## 13. 열린 질문 (설계 단계에서 확정)
- **캘린더 범위**: 단일 월(시안) → 임의 연·월 이동 + 과거 날짜 허용 정책(여행 시작일이 과거여도 되는지).
- **닫기 시 확인**: 입력 중 X/이탈 시 변경사항 폐기 확인 모달 노출 여부.
- **나라/지역 입력**: 자유 텍스트(시안) vs Google Places 자동완성 연동(구글맵 연동 설계와 함께).
