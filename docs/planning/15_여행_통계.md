# 15. 여행 통계 (이동거리 · 장소 분포)

> 완성된 일정을 바탕으로 여행의 요약 통계 — 총 이동거리, 일자별 이동, 카테고리별 장소 분포 — 를 차트로 보여준다. 데이터 대시보드 강점을 소비자 제품에 한 번 더 드러내는 화면. (2차 기획 — 신규 기능, 기능명세서 §2 "여행 통계(이동거리·장소 분포)" 근거)

## 1. 개요
- **화면 ID**: `trip-stats`
- **라우트**: `/trips/[id]?view=stats` (워크스페이스 5번째 뷰) — 또는 예산 뷰 내 탭 (구현 메모 §9 참조)
- **목적**: 일정 좌표·카테고리를 집계해 "이 여행이 어떤 여행이었는지"를 지표·차트로 요약한다. 팜플렛(14)의 "여행지 소개" 뒷면에도 재사용한다.

## 2. 사용자 & 진입 경로
- **사용자**: 여행 멤버(owner / editor / viewer 모두 열람 가능 — 읽기 지표)
- **진입**: 워크스페이스 뷰 세그먼트에 "통계" 추가(플랜/일정표/장소/예산/통계) · 팜플렛 화면에서 지표 재사용
- **이탈**: 다른 뷰 세그먼트 · "목록"

## 3. 구성 요소
> 시안: `docs/design/prototype/여행 통계.dc.html`(예정). 예산 대시보드(07)의 차트·카드 톤 재사용.

- **요약 카드(상단)**: 총 이동거리(km) · 총 장소 수 · 여행 일수 · 하루 평균 장소 수
- **일자별 이동거리 차트**: Day별 이동거리 막대(또는 라인) — 어느 날이 빡빡했는지
- **카테고리 분포 차트**: 카테고리(식당·카페·기념품·쇼핑·명소·숙소·교통·기타)별 장소 수 도넛/바 — 카테고리 토큰 색 사용
- **지역(area) 분포**(선택): 지역별 장소 수 목록
- **하이라이트 문구**(선택, AI): "가장 많이 간 카테고리", "가장 바쁜 날" 등 요약 카피

## 4. 데이터
- 일정 장소(itinerary): Day별 `lat/lng`·순서 → **이동거리 계산**(연속 장소 간 Haversine 합)
- 장소 `category` → 카테고리 분포 집계
- 장소 `area` → 지역 분포(선택)
- `Trip`(startDate·endDate) → 여행 일수
- **집계는 순수 셀렉터**로(서버 저장 안 함): `computeTripStats(places, trip)` → `{ totalDistanceKm, perDay[], byCategory[], byArea[] }`
- 클라 UI 상태: 없음(파생 데이터만). 서버 상태는 기존 `usePlacesQuery` 재사용

## 5. 상태
- **기본**: 지표·차트 렌더
- **빈 상태**: 일정 장소 0 → "일정을 추가하면 통계가 생겨요" 안내(차트 자리표시)
- **부분**: 좌표 없는 장소는 이동거리 계산에서 제외(주석 표기)
- **로딩**: 조회 중 카드·차트 스켈레톤
- **에러**: 조회 실패 시 재시도 안내(전역 패턴)

## 6. 인터랙션
1. 진입 → 일정 조회 → 집계 → 카드·차트 렌더
2. Day 필터(선택) → 특정 Day 지표만 강조
3. 카테고리 차트 hover → 개수·비율 툴팁
4. 팜플렛(14)에서 동일 셀렉터로 지표 임베드

## 7. 권한 분기
- 모든 멤버(owner/editor/viewer) 열람 가능(읽기 지표).
- 편집 동작 없음(파생 통계). 서버 접근은 기존 RLS(멤버만 조회)로 강제(§8.2).

## 8. 라우팅 & 연결
- **들어오기**: 워크스페이스 뷰 세그먼트 "통계"(§04) · 팜플렛(§14)
- **나가기**: 다른 뷰(`?view=plan|calendar|places|budget`) · "목록" → `/trips`

## 9. 구현 메모
- **배치 결정 필요**: (a) 워크스페이스 5번째 뷰 `?view=stats` 로 독립 / (b) 예산 뷰(07) 상단 탭으로 편입. 지표 성격이 예산과 다르므로 **(a) 독립 뷰 권장**(세그먼트에 "통계" 추가). — 설계 GATE에서 확정.
- **이동거리**: 연속 일정 좌표 간 Haversine 거리 합(직선). 실제 경로 거리(구글 Directions)는 후속/비용 고려.
- **차트**: Recharts 재사용(예산 대시보드와 동일 톤). 카테고리 색은 `lib/constants/category.ts` 단일 출처.
- **집계**: 순수 함수 `computeTripStats`(테스트 가능). 컴포넌트 직접 fetch 금지(§7.1), `usePlacesQuery` 재사용.
- **컴포넌트 매핑**: `features/stats`(요약 카드·차트·셀렉터), 공통 카드·차트 래퍼는 `components/ui`/기존 예산 컴포넌트 참고.
- **AI 하이라이트 문구**(선택): 서버 라우트에서 집계 결과로 요약 카피 제안(결정적 지표 + 제안 카피 분리).
- **팜플렛 연계**: 동일 셀렉터를 팜플렛(14) "여행지 소개" 뒷면에 재사용.

## 10. 디자인 시안 참조
- `docs/design/prototype/여행 통계.dc.html`(예정)
- 참고: 예산 대시보드(07) 시안의 카드·도넛·막대 톤

## 11. 수용 기준 (Acceptance Criteria)
- [ ] `?view=stats` 진입 시 요약 카드(총 이동거리·장소 수·일수·하루 평균)가 렌더된다.
- [ ] 일자별 이동거리 차트와 카테고리 분포 차트가 표시된다.
- [ ] 이동거리는 연속 일정 좌표 간 Haversine 합으로 계산되고, 좌표 없는 장소는 제외된다.
- [ ] 카테고리 색은 단일 출처 상수를 사용한다(하드코딩 0건).
- [ ] 일정 장소가 없으면 빈 상태 안내가 표시된다.
- [ ] 집계 로직(`computeTripStats`)에 단위 테스트가 있다.

**비기능**: 예산 대시보드와 시각 톤 일치 / 데스크톱 1280px / 차트 접근성(값 텍스트 제공).

## 12. 확정 사항 (MVP 범위)
- 이동거리 = **직선(Haversine) 합**(실제 경로 거리는 후속).
- 차트 = Recharts, 카테고리 색 단일 출처.
- 배치 = **독립 뷰 `?view=stats` 권장**(설계에서 확정).
- AI 하이라이트 문구는 **선택**(후속 가능).
