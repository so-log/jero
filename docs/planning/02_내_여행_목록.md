# 02. 내 여행 목록 (홈)

> 로그인 후 첫 화면. 내가 속한 여행들을 카드로 한눈에 보고, 새 여행을 시작하거나 워크스페이스로 진입한다.

## 1. 개요
- **화면 ID**: `trips-home`
- **라우트**: `/trips`
- **목적**: 내가 멤버인 여행 목록을 예정/지난으로 묶어 보여주고, 생성·진입의 허브 역할.

## 2. 사용자 & 진입 경로
- **사용자**: 로그인한 사용자 (자신이 멤버인 여행만 노출)
- **진입**: 로그인/가입 성공 직후 · 여행 생성(03) 완료 후 · 워크스페이스에서 "목록으로" · 전역 로고 클릭
- **이탈**: 카드 클릭 → 워크스페이스(`/trips/[id]`) · "새 여행 만들기" → `/trips/new` · 사용자 메뉴 → `/settings`

## 3. 구성 요소
> 레이아웃·색·간격은 시안 고정. 시안: `docs/design/prototype/내 여행 목록.dc.html`

**전역 헤더** (로그인 후 공통)
- 로고("jero", 클릭 시 `/trips`)
- 검색 입력("여행 또는 장소 검색")
- 알림 버튼(미확인 표시 점)
- 사용자 메뉴(아바타 이니셜 + 이름 + 펼침) → 계정 설정/로그아웃

**페이지 헤더**
- 제목 "내 여행" + 서브타이틀(요약: "N개의 여행 · 예정 N · 지난 N")
- "새 여행 만들기" 버튼 (빈 상태에서는 숨김 — 빈 상태 전용 CTA가 대체)

**필터/정렬 줄** (목록 있을 때만)
- 세그먼트 토글: **예정 / 지난 / 전체**
- 정렬 드롭다운: "최근 출발 순" (기준 변경 가능)

**여행 목록**
- 그룹 헤더(예: "예정된 여행" + 개수 배지)
- 3열 그리드의 **여행 카드**:
  - 커버(그라데이션 + 옅은 지도 그리드 + 대표 카테고리 아이콘 메달리언)
  - 커버 위 오버레이: **역할 배지**(소유자/편집자/뷰어, 프로스트 칩), **D-day 배지**(예정 여행만)
  - 제목, 기간 + N박N일
  - 멤버 아바타(최대 4 + "+N"), 장소 수("N곳")

## 4. 데이터
- `Trip` (id, title, startDate, endDate, coverColor/coverIcon)
- `Membership` (role) — **현재 사용자의 이 여행 내 역할** (카드 배지)
- 멤버 목록(아바타용): 각 멤버의 표시 이니셜·식별 색
- 집계: 장소 수(`Place` count)
- **파생값**(서버 또는 셀렉터): `nights`(기간에서), `dday`(시작일 − 오늘), `past`(종료일 < 오늘)
- 목록은 **요청자가 멤버인 Trip만** 반환(§8.2·§8.5 — 권한 없는 여행·타 멤버 과다 정보 비노출)

## 5. 상태
시안 상태 스위처(`list` / `empty` / `loading`)에 대응.
- **목록(list)**: 필터에 따라 예정/지난 그룹으로 묶어 카드 그리드 표시
- **빈 상태(empty)**: 일러스트 + "아직 떠날 여행이 없어요" 안내 + "첫 여행 만들기" CTA (헤더의 생성 버튼·필터 줄 숨김)
- **로딩(loading)**: 카드 스켈레톤 그리드(셔머)
- **에러**: 목록 로드 실패 안내(재시도) — 시안 외, 전역 패턴 적용
- 필터별 빈 그룹: 해당 그룹은 렌더하지 않음(예: "지난" 없으면 그룹 생략)

## 6. 인터랙션
1. 세그먼트(예정/지난/전체) 선택 → 표시 그룹 즉시 전환 (URL 쿼리 동기화 권장 — 공유·복원)
2. 정렬 변경 → 카드 정렬 갱신
3. 검색 입력 → 여행/장소 필터링(범위는 §11 확정)
4. 카드 클릭 → 해당 여행 워크스페이스(`/trips/[id]`, 기본 플랜 뷰)
5. "새 여행 만들기" / "첫 여행 만들기" → `/trips/new`
6. 사용자 메뉴 → 계정 설정/로그아웃, 알림 버튼 → 알림(범위 §11)

## 7. 권한 분기
- 목록 자체는 로그인 사용자 모두 동일하게 "내 여행"을 본다(역할별 화면 차이 없음).
- 각 카드의 **역할 배지**가 그 여행에서의 내 권한(owner/editor/viewer)을 표시 — 실제 편집 권한 강제는 워크스페이스/서버에서(§8.2).
- 비로그인 접근 시 이 라우트는 보호됨 → `/`로 리다이렉트.

## 8. 라우팅 & 연결
- **들어오기**: 로그인 성공 · 여행 생성 완료 · 워크스페이스 "목록으로" · 로고
- **나가기**:
  - 카드 → `/trips/[id]` (워크스페이스, §04~07)
  - 새/첫 여행 만들기 → `/trips/new` (§03)
  - 사용자 메뉴 → `/settings` (§09)
- 시안 내 네비게이션(참고): 카드 → `플랜 뷰.dc.html`, 생성 버튼 → `새 여행 만들기.dc.html`, 사용자 메뉴(지현) → `계정 설정.dc.html`

## 9. 구현 메모
- 데이터: `features/trip/api`의 TanStack Query `useTripsQuery`(목록). 컴포넌트 직접 fetch 금지(§7.1).
- 클라 상태: 필터(예정/지난/전체)·정렬은 URL `searchParams`로 동기화(Zustand 불필요).
- 파생값(`nights`/`dday`/`past`): 가능한 서버 응답에 포함하거나 순수 셀렉터(`lib`)로 계산 — `dday`는 "오늘" 기준 시점 의존이므로 클라 계산 시 타임존 주의.
- 컴포넌트 매핑: `features/trip/components`(`TripCard`, `TripList`, 필터/정렬), 전역 헤더는 레이아웃 공통(`app/(app)/layout.tsx` + `components/ui`).
- 역할 배지·아바타 묶음은 재사용 컴포넌트(`components/ui` 또는 `features/member`) — 04 등 다른 화면과 공유.
- 디자인 토큰(커버 그라데이션·역할 색·radius)은 `support.js`에서 추출해 `@theme` 반영(하드코딩 금지, §7.1).
- 빈/로딩/에러 상태 컴포넌트는 전역 공통 패턴으로.

## 10. 디자인 시안 참조
- `docs/design/prototype/내 여행 목록.dc.html` (상태: list/empty/loading)

## 11. 수용 기준 (Acceptance Criteria)
> 이 화면이 "done"인 조건. GATE 1 승인·QA 테스트 시나리오의 근거. (체크리스트 = 검증 항목)

- [ ] 로그인 사용자가 `/trips` 진입 시 **자신이 멤버인 여행만** 카드로 표시되고, 타인의 여행은 노출되지 않는다(§8.2).
- [ ] 목록(list)/빈(empty)/로딩(loading) 상태가 시안 스위처대로 렌더된다 — 빈 상태는 생성 버튼·필터 줄을 숨기고 전용 CTA를 보이며, 로딩은 카드 스켈레톤을 표시한다.
- [ ] 세그먼트(예정/지난/전체) 전환 시 표시 그룹이 즉시 바뀌고, URL 쿼리에 동기화되어 새로고침·공유 시 상태가 복원된다.
- [ ] 빈 그룹(예: "지난" 여행 없음)은 렌더되지 않는다.
- [ ] 각 카드에 역할 배지(소유자/편집자/뷰어), 예정 카드의 D-day, 기간(N박N일), 멤버 아바타(최대 4 + "+N"), 장소 수가 표시된다.
- [ ] 카드 클릭 시 해당 여행 워크스페이스(`/trips/[id]`, 기본 플랜 뷰)로 이동한다.
- [ ] "새 여행 만들기" / 빈 상태 "첫 여행 만들기" 클릭 시 `/trips/new`로 이동한다.
- [ ] 비로그인 상태로 `/trips` 접근 시 `/`로 리다이렉트된다.
- [ ] 목록 로드 실패 시 재시도 가능한 에러 안내가 표시된다.

**비기능**: 데스크톱 1280px 기준 시안과 시각 일치 / 3열 그리드 반응형 / `dday`·`past` 계산이 타임존 기준으로 정확.

## 12. 확정 사항 (MVP 범위)
- **검색 범위**: ✅ 여행 제목 + 장소명(시안 카피 "여행 또는 장소 검색"에 맞춤). 클라이언트 필터(로드된 목록 대상).
- **알림**: ❌ MVP 제외. 헤더 알림 버튼은 비활성(미확인 점 표시 없음, 후속 과제).
- **커버 이미지**: ✅ 그라데이션 + 카테고리 아이콘 **고정**. 사용자 업로드 미지원 → `Trip` 스키마는 `coverColor`/`coverIcon`만(이미지 URL 컬럼 없음).

## 13. 열린 질문 (설계 단계에서 확정)
- **정렬 옵션**: "최근 출발 순" 외 제공 항목(생성순/이름순 등).
- **D-day/기간 계산 위치**: 서버 응답 포함 vs 클라 계산(타임존 기준).
