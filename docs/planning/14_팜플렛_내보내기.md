# 14. 팜플렛 내보내기 (여행 팜플렛)

> 친구들과 정한 루트·일정을 **A4 3단 접이 팜플렛(PDF)**으로 내보낸다. 표지는 여행 분위기에 맞춘 테마로, 안쪽은 일정, 뒷면은 준비물·여행지 소개·QR(공유 링크)로 구성한다. (2차 기획 — 신규 기능)

## 1. 개요
- **화면 ID**: `pamphlet-export`
- **라우트**: `/trips/[id]/pamphlet`
- **목적**: 완성된 여행 일정을 인쇄·공유 가능한 팜플렛(PDF)으로 내보낸다. 포함 섹션을 체크로 선택하고, 여행 정보 기반 테마로 디자인해 결과물의 완성도를 높인다.

## 2. 사용자 & 진입 경로
- **사용자**: 여행 멤버(owner / editor). (viewer·유료 게이팅은 §7·후속)
- **진입**: 워크스페이스 상단 더보기(⋯) "팜플렛 만들기" · 공유 오버레이(#share) "팜플렛으로 내보내기"
- **이탈**: 내보내기 완료(PDF 다운로드) 후 워크스페이스 복귀 · "닫기"

## 3. 구성 요소
> 레이아웃·색은 시안 기준. 여기서는 요소와 동작만 정의. 시안: `docs/design/prototype/팜플렛.dc.html`(예정)

**좌측 — 설정 패널**
- **포함 섹션 체크박스**: 표지 / 일정표 / 준비물 / 여행지 소개 / QR (표지·QR 기본 on)
- **테마 선택**: 프리셋 칩 — 색 팔레트 + 커버 스타일. ① 차분한 패턴 계열(여름 바다 · 도시 · 벚꽃 · 설경 · 미식 · 기본) ② **일러스트 씬 계열(밝고 귀여운 게임느낌 — 해·하늘 / 잔디 들판 / 숲 / 바다 / 기차 / 비행기 등, 표지 배경에 플랫 일러스트 씬)**. 여행 분위기에 맞춰 톤을 고른다. (씬 테마는 Claude Design 시안 확정 후 프로토타입·상수에 반영)
- **AI 추천 버튼**: 여행 정보(나라·지역·기간·계절·카테고리 분포)로 어울리는 테마 + 커버 문구 제안(적용은 사용자 선택)
- **준비물 편집**(준비물 섹션 on일 때): 체크리스트 항목 추가/삭제(자유 입력)
- **내보내기 버튼**: PDF 생성·다운로드 (생성 중 진행 표시)

**우측 — 미리보기**
- **A4 3단 미리보기**: 앞면(3면) + 뒷면(3면). 접힘 순서대로 배치된 실측 비율 미리보기
- 섹션 토글·테마 변경 시 즉시 반영

**팜플렛 면 구성(기본 접이 순서)**
- 표지면: 템플릿 제목 + 기간 + 테마 디자인
- 일정면(1~2면): Day별 일정 표(순번·시간·장소·카테고리·메모)
- 뒷면: 준비물 체크리스트 / 여행지 소개(요약) / **QR(공유 링크)**

## 4. 데이터
- `Trip`(title, country, region, startDate, endDate) — 표지·기간
- `Place`/일정(itinerary) — Day별 장소·순서·시간·카테고리·메모(일정표 섹션)
- `ShareLink.token`(기존) — QR 대상 `/share/[token]`(읽기 전용). 없으면 발급(§8.2 스코프·만료)
- **테마 프리셋**(단일 출처 상수): `{ key, label, palette, coverStyle }` — `lib/constants/pamphletThemes.ts`
- **AI 추천 결과**: `{ themeKey, coverCopy }` — 서버 라우트 응답(저장 안 함, 제안용)
- **준비물 항목**: 클라 상태(체크리스트, 여행별 저장 여부는 후속)
- **선택 섹션 상태**: `{ cover, itinerary, checklist, intro, qr }` (클라 UI 상태)

## 5. 상태
- **기본(편집)**: 섹션 선택·테마 변경·미리보기·내보내기 가능
- **빈 상태**: 일정 장소 0 → 일정표 섹션 비활성 + 안내("일정을 추가하면 팜플렛에 담을 수 있어요")
- **로딩**: 여행·일정 조회 중 스켈레톤
- **생성 중**: PDF 생성 진행 표시(버튼 disabled + 스피너)
- **AI 추천 중**: 추천 버튼 로딩 → 결과 제안 카드
- **에러**: 조회/생성/추천 실패 시 재시도 안내

## 6. 인터랙션
1. 진입 → 여행·일정 조회 → 기본 테마·기본 섹션으로 미리보기 렌더
2. 섹션 체크 토글 → 미리보기 즉시 반영(면 구성 재계산)
3. 테마 칩 선택 → 색·커버 스타일 갱신
4. "AI 추천" → 여행 정보로 테마+커버 문구 제안 → 사용자가 적용/무시
5. (준비물 on) 체크리스트 항목 추가/삭제
6. "PDF 내보내기" → A4 3단 규격 PDF 생성·다운로드
7. QR: 공유 링크 없으면 발급 후 QR 생성(읽기 전용 스코프)
8. "닫기" → 워크스페이스 복귀

## 7. 권한 분기
- **owner / editor**: 팜플렛 생성·내보내기 가능.
- **viewer**: 기본 비노출(후속에서 "공유받은 플랜도 내보내기" 허용 검토).
- QR은 **읽기 전용 공유 토큰**만 사용(민감 필드·예산 제외, §8.2·08 공유 규약 준수).
- **유료화 게이팅(후속)**: 무료=미리보기/워터마크, 유료=고품질 PDF 등. MVP는 게이팅 없이 기능만.

## 8. 라우팅 & 연결
- **들어오기**: 워크스페이스 더보기(⋯)(§04) · 공유 오버레이(§10 #share)
- **나가기**: PDF 다운로드 후 잔류 · "닫기" → `/trips/[id]?view=plan`
- QR 대상: `/share/[token]`(§08 뷰어 공유) 재사용

## 9. 구현 메모
- **PDF 생성**: A4(210×297mm) 3단 접이 규격. 3패널 폭 배분(접힘 여백 고려: 안쪽 접히는 면을 약간 좁게). 서버 라우트에서 HTML→PDF 렌더 권장(폰트·색 일관). print CSS는 후순위.
- **테마**: 프리셋을 **단일 출처 상수**로(색·커버 스타일). 하드코딩 금지(§7.1). Tailwind `@theme` 토큰과 매핑.
- **AI 추천**: 서버 라우트(`POST /api/pamphlet/theme-suggest`)에서 여행 메타로 테마 키 + 커버 카피 제안. 렌더는 결정적(제안만 AI).
- **QR**: 경량 QR 라이브러리로 공유 링크 인코딩. 링크 없으면 `issueShareLink` 재사용해 발급.
- **데이터**: 컴포넌트 직접 fetch 금지(§7.1) — `features/pamphlet/api`(여행·일정은 기존 쿼리 재사용). 섹션·테마·준비물은 로컬/Zustand UI 상태.
- **컴포넌트 매핑**: `features/pamphlet`(설정 패널·미리보기·면 컴포넌트·테마), 공통 칩·체크박스·모달은 `components/ui`.
- **보안**: QR은 읽기 전용 토큰 스코프·만료(§8.2). 서버 라우트는 멤버십·역할 재검증.
- **의존 기능**: 여행 통계(§E, 이동거리·장소 분포)를 뒷면 "여행 소개"에 재사용 가능(선택).

## 10. 디자인 시안 참조
- `docs/design/prototype/팜플렛.dc.html`(예정) — 설정 패널 + A4 3단 미리보기 + 테마 프리셋 예시
- 참고: 첨부 레퍼런스(지역 여행 3단 리플렛)의 정보 밀도·동선 표현

## 11. 수용 기준 (Acceptance Criteria)
- [ ] 워크스페이스에서 팜플렛 화면 진입 시 여행·일정 기반 미리보기가 A4 3단 비율로 렌더된다.
- [ ] 섹션 체크박스(표지/일정표/준비물/여행지 소개/QR)로 포함 항목을 선택하면 미리보기가 즉시 반영된다.
- [ ] 테마 프리셋을 바꾸면 색·커버 스타일이 갱신된다.
- [ ] "AI 추천" 시 여행 정보 기반 테마·커버 문구가 제안되고, 적용/무시할 수 있다.
- [ ] "PDF 내보내기" 시 A4 3단 규격 PDF가 생성·다운로드되고, 선택한 섹션만 포함된다.
- [ ] QR은 읽기 전용 공유 링크(`/share/[token]`)로 연결되며, 링크가 없으면 발급된다.
- [ ] 일정 장소가 없으면 일정표 섹션이 비활성되고 안내가 표시된다(빈 상태).
- [ ] viewer에게는 내보내기 UI가 노출되지 않는다(권한 분기). 서버에서도 재검증한다.

**비기능**: A4 인쇄 시 접이 경계·여백이 어긋나지 않음 / 미리보기와 PDF 결과가 시각적으로 일치 / 데스크톱 1280px 기준.

## 12. 확정 사항 (MVP 범위)
- 디자인 생성 = **테마 프리셋 + AI 추천**(자유 이미지 생성은 Phase 2).
- 출력 = **PDF 내보내기**(print CSS 후순위).
- QR = **기존 공유 링크 재사용**(별도 딥링크 아님).
- 섹션 = 표지 · 일정표 · 준비물 · 여행지 소개 · QR (체크 선택).
- 유료화 게이팅은 **후속**(MVP는 기능만).
