# 17. 다중 도시 (Multi-City Trip)

> 한 여행에 **여러 도시**(예: 도쿄 3일 → 오사카 2일)를 담고, 도시별 일정·장소를 **한 여행 안에서 통합 조망**하는 기능.
> 차별점: 트리플의 검증된 약점 — "도시별로만 일정, 여러 도시를 한 번에 못 봄" — 을 정면 공략.

## 1. 화면 목적
- 여행을 **도시 세그먼트의 나열**로 구성한다(도쿄 → 오사카 → 교토).
- 각 도시에 기간(박)·장소·일정을 두되, **하나의 여행에서 전체를 조망**하고 도시 간 전환한다.
- **기존 단일 도시 여행은 그대로 동작**(도시 1개짜리 = 현재와 동일 경험).

## 2. 사용자 · 진입 경로
- **생성 마법사**: "여행 정보"에서 도시를 **여러 개 추가**(도시명 + 박수). 날짜는 도시 순서·박수로 자동 배분.
- **플랜/캘린더/장소 뷰**: 도시 컨텍스트로 조망·전환.
- **권한**: 도시 추가/편집/순서변경/삭제는 `owner`/`editor`. `viewer`는 조회만. 서버(RLS)에서 강제.

## 3. 구성 요소
- **도시 목록·순서**: 여행의 도시들을 순서대로(1. 도쿄 3박 / 2. 오사카 2박). 순서 변경(드래그).
- **도시별 기간**: 박수 → 날짜 범위 자동(도쿄 4/8–4/11, 오사카 4/11–4/13). 합 = 여행 전체 기간.
- **플랜 상단 도시 전환**: "도쿄 · Day 2" 처럼 현재 Day가 속한 도시 표시 + 도시 점프.
- **도시 간 이동(선택)**: 세그먼트 경계에 항공/기차 등 이동 표시(초기엔 transport 카테고리 장소로 대체 가능).
- **지도**: 현재 도시 중심으로 이동(도시들이 멀리 떨어져 한 지도에 다 담기 어려움 → 도시별 뷰).

## 4. 데이터 · 상태
- **모델**: `trip 1─N city 1─N place`. 여행이 도시들을 갖고, 장소는 도시에 속함.
- **도시**: 이름·국가·순서(sequence)·박수(nights) → 날짜 범위 파생.
- **Day→도시 매핑**: 각 날짜가 어느 도시 범위에 드는지로 결정(순서+박수로 계산).
- **하위호환**: 기존 `trip.country/region` 단일 → 마이그레이션으로 **도시 1개**로 이관. 앱은 도시 배열을 보되, 1개면 현재와 동일.

## 5. 인터랙션 (플로우)
1. **생성**: 마법사에서 도시 추가(도쿄 3박, 오사카 2박) → 총 5박 6일 자동, 각 도시 날짜 범위 표시.
2. **플랜**: Day 스위처가 도시 경계를 인지("도쿄 Day 3" 다음 "오사카 Day 1"). 지도는 도시 중심 이동. 장소·동선은 그 도시 것만.
3. **장소 저장**: 장소를 특정 도시에 배정(기본=현재 보고 있는 도시). 도시별 필터.
4. **도시 편집**: 박수 변경 시 날짜 재배분(영향받는 일정 경고). 도시 삭제 시 그 도시 장소 처리(미배정/삭제 확인).

## 6. 제약 · 엣지 케이스
- **도시 0/1개**: 1개면 현재와 동일(단일 도시 여행). 0개 방지(최소 1).
- **기간 정합**: 도시 박수 합 = 여행 전체 일수. 마법사에서 검증.
- **박수 변경 → 날짜 재배분**: 뒤 도시 날짜 밀림 → 배정된 일정이 범위를 벗어나면 경고(동선 최적화 B3의 "범위 밖 미배정" 패턴 재사용).
- **도시 삭제**: 그 도시 장소를 미배정 처리 or 함께 삭제(ConfirmDialog).
- **좌표 없는 도시**: 지도 중심 못 잡으면 이전 도시/기본 중심 유지.
- **동선 최적화**: 하루(=한 도시 내) 단위라 다중 도시와 **충돌 없음**(도시 간 최적화는 범위 밖).

## 7. 권한
- 도시 CRUD·순서: `owner`/`editor`. `viewer` 조회. 서버(RLS `trip_city`) 강제.

## 8. 라우팅
- 별도 라우트 없음. 기존 플랜/캘린더/장소 뷰에 도시 컨텍스트 추가. (도시 선택은 뷰 내 상태 or `?city=` 쿼리 선택적.)

## 9. 트리플 대비 우위
| | 트리플 | jero |
|---|---|---|
| 다도시 | ❌ 도시별 분리, 한 번에 못 봄 | ✅ **한 여행에 여러 도시 통합 조망** |
| 도시 순서·기간 | — | ✅ 순서·박수 → 날짜 자동 배분 |
| 도시 전환 | 별도 일정 | ✅ 플랜에서 도시 넘나들며 조망 |

## 10. 수용 기준
- [ ] 마법사에서 도시 2개+ 추가 → 박수 합이 여행 일수와 맞고, 각 도시 날짜 범위가 자동 계산된다.
- [ ] 생성된 여행의 플랜에서 Day가 도시별로 그룹/전환되고, 지도는 해당 도시 중심으로 이동한다.
- [ ] 장소가 도시에 배정되고, 도시별로 필터/조망된다.
- [ ] **기존 단일 도시 여행이 그대로 동작**한다(도시 1개로 이관, 회귀 없음).
- [ ] 도시 박수 변경 시 날짜 재배분 + 범위 밖 일정 경고.
- [ ] 도시 삭제 시 장소 처리 확인.
- [ ] `viewer`는 도시 편집 불가(서버 거부).
