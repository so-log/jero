# 16. 동선 최적화 (Route Optimization)

> 하루 일정의 방문 순서를 **총 이동을 최소화**하도록 자동 정렬하는 기능.
> 차별점: 트리플의 단순 자동정렬(초기 지점 기준 근접순)을 넘어 **전체 경로 최적화(NN + 2-opt) + 실제 이동시간 + 시작/끝 앵커 제약**.

## 1. 화면 목적
- 사용자가 하루에 담은 장소들의 **방문 순서를 최적(최소 이동거리/시간)으로 재배치**해준다.
- 수동 드래그(기존)와 병행: "직접 정렬" + "자동 최적화" 둘 다 제공.
- 적용 전 **미리보기**(새 순서·동선·총 이동시간 전후)로 확인 후 반영, 되돌리기 가능.

## 2. 사용자 · 진입 경로
- **진입**: 플랜 뷰(`?view=plan`)의 특정 Day 일정 패널.
- **트리거**: Day 헤더/툴바의 **"동선 최적화"(또는 "최적 순서로 정렬")** 버튼.
- **권한**: `owner`/`editor`만(순서 변경=쓰기). `viewer`는 버튼 미노출. 서버는 `reorder_places`(RLS editor+)로 재확인.

## 3. 구성 요소
- **최적화 버튼**: Day별. 2곳 미만이면 비활성(정렬할 게 없음).
- **결과 미리보기**: 새 순서 리스트 + 지도 동선(Polyline)·번호 마커 프리뷰 + **총 이동시간/거리 전 → 후**(예: "3.2시간 → 1.8시간").
- **적용 / 취소** 버튼. 적용 후 **되돌리기(undo)**(직전 순서 복원).
- **옵션(선택)**:
  - **시작/끝 앵커**: 첫·마지막을 고정(예: 숙소에서 출발→숙소로 복귀).
  - **이동수단**: 도보 / 대중교통(실이동시간 계산 시).

## 4. 데이터 · 상태
- **입력**: 그 Day의 배정 장소(`scheduled_date`=해당일) 중 **좌표(lat/lng) 있는 것**. 좌표 없는 장소는 최적화 대상에서 제외(경고 표시, 순서 유지).
- **중간**: 장소 간 이동 비용 매트릭스(거리 또는 실이동시간) → 최적 순서(배열).
- **출력**: 재배치된 `order_in_day`. 저장은 기존 `reorder_places(trip_id, date, ids[])` RPC 재사용(신규 스키마 불필요).
- **상태**: 계산·미리보기는 로컬(클라) 상태. 적용 시에만 서버 반영(낙관적 업데이트).

## 5. 인터랙션 (플로우)
1. "동선 최적화" 클릭 → 계산(로딩 스피너, 보통 즉시).
2. 미리보기 표시: 새 순서·동선·"총 이동 전→후".
3. **적용** → `useReorderPlaces`로 순서 저장(낙관적) + 지도/리스트 갱신. **취소** → 원복.
4. 적용 직후 **되돌리기** 노출(직전 순서로).
- 좌표 없는 장소가 있으면: "N개 장소는 위치가 없어 최적화에서 제외됐어요"(순서 유지).
- 앵커 지정 시: 첫/끝 고정하고 중간만 최적화.

## 6. 제약 · 엣지 케이스
- **장소 < 2** → 버튼 비활성.
- **좌표 없는 장소** → 제외 + 안내(수기 입력 장소 등).
- **실이동시간 API 실패/쿼터 초과** → 직선거리(Haversine)로 폴백(경고 없이 자연스럽게).
- **앵커가 좌표 없음** → 앵커 무시(경고).
- **동일 위치 다수** → 순서 안정성 유지(불필요한 재배치 최소화).

## 7. 권한
- 버튼·적용: `owner`/`editor`. `viewer` 미노출 + 서버(RLS `reorder_places`) 강제.
- UI 숨김은 UX일 뿐, 실제 차단은 서버.

## 8. 라우팅 연결
- 별도 라우트 없음. 플랜 뷰(`/trips/[id]?view=plan`) 내 인라인/오버레이.

## 9. 트리플 대비 우위 (차별 포인트)
| | 트리플 | jero(이 기능) |
|---|---|---|
| 정렬 방식 | 자동 정렬(기본, 근접순 추정) | **NN + 2-opt 전체 경로 최적화** |
| 비용 기준 | (불명확) | **실제 이동시간(Directions)** + 직선거리 폴백 |
| 제약 | — | **시작/끝 앵커**(숙소 출발·복귀) |
| 투명성 | 결과만 | **전/후 총 이동시간 비교** 제시 |
| 수동 병행 | — | 드래그 수동 정렬과 **공존** |

## 10. 수용 기준 (검증 체크리스트)
- [ ] 좌표 있는 장소 2곳+ 인 Day에서 버튼 활성, 미만이면 비활성.
- [ ] 클릭 → 미리보기에 새 순서·동선·총 이동시간 전후가 표시된다.
- [ ] 적용 시 리스트·번호 마커·지도 동선이 새 순서로 갱신되고 서버(`reorder_places`)에 저장된다.
- [ ] 되돌리기로 직전 순서 복원.
- [ ] 좌표 없는 장소는 제외되고 안내가 뜬다.
- [ ] API 실패 시 직선거리 폴백으로도 정렬된다(크래시 없음).
- [ ] `viewer`는 버튼이 안 보이고, 서버가 순서 변경을 거부한다.
- [ ] 최적화 결과의 총 이동거리가 원래 순서보다 **작거나 같다**(2-opt 개선 보장).
