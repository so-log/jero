# 여행 플래너 (협업 여행 가이드북) — 기능 명세서 v0.1

> 포트폴리오 메인 프로젝트. **"실시간 협업 + 데이터 대시보드가 들어간 여행 플래너"**로,
> 소비자용 제품의 재미와 어드민/데이터 UI 강점을 동시에 보여주는 것이 목표.

작성일: 2026-06-23 · 상태: 초안 (논의용)

---

## 1. 제품 개요

### 한 줄 소개
친구들과 **함께** 여행 일정을 짜고, 순서대로 동선을 그리고, 예산을 정산하는 **협업 여행 플래너**.

### 누구를 위한 것인가
- 2~5명이 함께 가는 여행을 계획하는 그룹 (제주·일본 같은 국내외 단기 여행)
- "구글맵에 핀만 잔뜩 찍어두고 정작 순서·동선·역할이 정리 안 되는" 경험을 한 사람

### 구글맵으로는 안 되는 것 (차별점)
| | 구글맵 저장 | 우리 제품 |
|---|---|---|
| 위치 저장 | O | O (폴더별 저장) |
| **순서 지정** | X | O (드래그앤드롭) |
| **순서대로 동선 표시** | X | O |
| **여러 명이 동시 편집** | X | O (실시간) |
| **권한(수정/읽기) 공유** | 제한적 | O |
| **예산·정산** | X | O (대시보드) |
| **일정표(월/주/일)** | X | O |

핵심 한 문장: **"위치를 저장만 하는 게 아니라, 순서·동선·예산·역할까지 함께 굴리는 여행 계획서."**

## 2. 포트폴리오 목표와의 연결 (왜 이 기능들인가)

이 프로젝트는 단순한 여행앱이 아니라, **경력의 강점을 소비자 제품 위에 얹어 증명**하는 게 목적이다.
각 기능이 어떤 강점/스택을 보여주는지 의도적으로 매핑한다.

| 보여줄 강점 (경력) | 이 제품에서 드러나는 기능 |
|---|---|
| 대량 데이터 테이블·필터·정렬·**가상화** | 일정표(월/주/일), 저장 장소 목록, "오늘만 보기" 필터 |
| 통계 **대시보드·차트** | 예산/정산 대시보드, 여행 통계(이동거리·장소 분포) |
| **승인/변경 워크플로우** (결재선 경험) | 권한(소유자/편집/뷰어), 변경 이력, 제안·코멘트 |
| 사용자/권한·설정, 모달·다단계 폼 | 템플릿 생성 마법사(다단계 폼), 공유·초대 |
| **실시간** (socket.io 경험) | 동시 편집, 다른 사람 커서 표시 |
| 최신 스택 학습 (목표) | Next.js 15 + Zustand + TanStack Query/Table/Virtual + Tailwind/shadcn |

> README에서 강조할 포인트: "데이터 밀집 어드민 UI를 4년 해온 개발자가, 같은 역량을
> 소비자 협업 제품에 적용했을 때 어떻게 풀어내는가."

## 3. MVP 범위 (핵심만 탄탄하게)

> **2차 구현 현황 (2026-07-08 갱신)**: 1차 구현으로 MVP 대부분이 실동작한다(인증·CRUD·공유·실시간·예산). 아직 미구현·부분구현으로 남은 6개 항목(실시간 커서 · 폴더 관리 · 템플릿 복제 · 구글 OAuth · 여행 통계 · 메모 자동저장)과 신규 기능(팜플렛 내보내기)은 2차 범위다. 구현 갭 상세는 [`docs/remaining-features.md`](../remaining-features.md), 구현 설계는 [`docs/architecture/2차_구현_설계.md`](../architecture/2차_구현_설계.md), 신규 기획은 `docs/planning/14`·`15`를 참조한다.

### ✅ MVP에 포함 (이번에 끝까지 만든다)
1. 인증 / 로그인
2. 여행(템플릿) 생성 — 다단계 폼 (제목·아이콘·나라·지역·기간·함께하는 사람·권한)
3. 지도 + 장소 추가 / **폴더별 저장**
4. **순서 지정(드래그앤드롭) + 순서대로 동선 표시**
5. 일정표 — 월/주/일 뷰, 일자별 상세, 장소 카테고리별 아이콘
6. **필터** — "오늘 일정만", "저장 장소만" 등 지도 표시 토글
7. 공유 / **권한**(소유자·편집자·뷰어) + 초대
8. **실시간 동시 편집** (커서/변경 동기화)
9. 장소별 메모
10. **예산 / 정산 대시보드** (카테고리별 지출, 1인당 더치페이, 차트)

### 🕒 Phase 2 (README에 "향후 계획"으로 명시, 지금은 안 만듦)
- AI 챗봇 (지역 특산물·기념품 추천)
- 여행 일기 / 하루 소감 + 발자국(이동거리) 기록
- 사진 기록 / 셀로그식 콜라주
- 투어 등록·연계 (일정표 자동 채우기)
- 변경 이력 타임라인 (시간 되면 MVP로 끌어올려도 좋음)

### 📱 Phase 3 (모바일 앱 전용)
- GPS 자동 발자국 기록
- 영수증 OCR → 가계부 자동 입력
- 사진 EXIF 자동 장소 분류
- 푸시 알림 / 오프라인 모드 / 환율 계산기 / 음성 메모 / 홈 위젯

> 원칙: **"적게, 끝까지, 폼나게."** 절반 만든 화려한 기능보다, 완성도 높은 핵심 10개가 채용에 강하다.

## 4. 핵심 기능 상세 명세

각 기능: **설명 / 주요 동작·화면 / 데이터 / 보여줄 기술 포인트** 순으로 정리.

### 4.1 인증 / 로그인
- 설명: 이메일 또는 소셜(구글) 로그인. 비로그인 시 읽기 전용 공유 링크는 열람 가능.
- 데이터: User(id, name, email, avatar)
- 기술 포인트: 라우트 보호, 세션 처리. (NextAuth 등)

### 4.2 여행 생성 마법사 (다단계 폼)
- 설명: "새 여행 만들기" → 단계별 입력. 첫 화면에서 여행의 골격을 잡는다.
- 주요 동작:
  - Step 1: 제목 + 아이콘/커버 (예: 제주도 🏝️)
  - Step 2: 나라 / 지역 / 기간(시작~종료일)
  - Step 3: 함께하는 사람 초대 (이메일/링크) + 각자 권한(편집/읽기)
  - Step 4: 빈 일정 vs 템플릿 복제로 시작
- 데이터: Trip(id, title, icon, country, region, startDate, endDate, ownerId, members[])
- 기술 포인트: **React Hook Form + Zod 다단계 폼**, 단계별 검증, 진행 상태 관리. (경력의 "모달·다단계 폼" 강점)

### 4.3 지도 + 장소 추가 / 폴더별 저장
- 설명: 지도에서 장소를 검색·핀 추가. 가고 싶은 곳을 **폴더로 분류**해 저장 (예: "맛집", "카페", "쇼핑"). 저장 ≠ 일정 등록.
- 주요 동작:
  - 검색 → 핀 추가 → 폴더 지정
  - 저장 장소는 지도에 한 종류 마커, 일정에 등록된 장소는 다른 마커 + 직선 연결
  - 장소 클릭 → 상세(메모, 카테고리, 일정에 추가 버튼)
- 데이터: Place(id, tripId, name, lat, lng, category, folderId, memo), Folder(id, tripId, name)
- 기술 포인트: 지도 SDK 연동, 마커 클러스터링, 상태 동기화. **지도는 Google Maps로 확정**(해외 여행까지 커버 — 글로벌 장소 데이터·길찾기).

### 4.4 순서 지정 + 동선 표시 ⭐
- 설명: 하루 일정 안에서 장소들을 **드래그앤드롭으로 순서 변경**. 순서대로 지도에 동선(경로)을 그린다.
- 주요 동작:
  - 일정 패널에서 장소 카드 드래그 → 순서 재배치
  - 순서 바뀌면 지도 경로 즉시 갱신
  - (Phase 2) 장소 간 이동 시간 자동 계산 → 무리한 일정 경고
- 데이터: ItineraryItem(id, tripId, date, order, placeId, startTime?, note?)
- 기술 포인트: **드래그앤드롭 + 정렬 상태 관리(TanStack/dnd-kit)**, 낙관적 업데이트. 어드민의 "정렬·순서" UX 역량 직결.

### 4.5 일정표 (월 / 주 / 일 뷰) ⭐
- 설명: 여행 일정을 월·주·일 단위로 전환해 본다. 일 단위 상세에서 장소별 **카테고리 아이콘**(식당/카페/기념품/쇼핑/박물관 등)으로 구분.
- 주요 동작:
  - 뷰 토글: 월 → 주 → 일
  - 일 상세: 타임라인 형태로 장소 카드 나열, 아이콘·메모 표시
  - 가상화로 긴 목록도 매끄럽게
- 데이터: 4.4의 ItineraryItem을 날짜로 그룹핑
- 기술 포인트: **TanStack Table/Virtual로 데이터 밀집 뷰 + 뷰 전환**. 경력의 "테이블·필터·가상화" 강점이 가장 잘 드러나는 화면.

### 4.6 필터 / 지도 표시 토글
- 설명: "오늘 일정만 보기" 체크 시 저장 장소 마커는 숨김. 폴더·카테고리·날짜로 필터.
- 주요 동작: 필터 칩/체크박스 → 지도·목록 동시 반영
- 기술 포인트: 다중 필터 상태 조합, URL 쿼리 동기화(공유 가능한 필터 상태).

### 4.7 공유 / 권한 / 초대 ⭐
- 설명: 여행을 사람별로 **소유자 / 편집자 / 뷰어** 권한으로 공유. 초대 링크 발급.
- 주요 동작:
  - 멤버 관리 모달: 권한 변경, 초대, 제거
  - 뷰어는 읽기만, 편집자는 일정 수정 가능
- 데이터: Membership(tripId, userId, role: owner|editor|viewer)
- 기술 포인트: **권한 기반 UI 분기**(버튼 비활성/숨김), 초대 플로우. 경력의 "사용자/권한·설정" 직결.

### 4.8 실시간 동시 편집 ⭐
- 설명: 여러 명이 같은 여행을 동시에 편집할 때, 변경이 실시간 반영되고 다른 사람의 커서/선택이 보인다.
- 주요 동작:
  - A가 장소 순서 바꾸면 B 화면에 즉시 반영
  - 현재 접속 중인 멤버 아바타 표시 (presence)
- 기술 포인트: **WebSocket/실시간 동기화(socket.io 경험)**, presence, 충돌 처리. 가장 강한 차별화 포인트.

### 4.9 장소별 메모
- 설명: 각 장소에 자유 메모 (예약 정보, 추천 메뉴, 주의사항).
- 데이터: Place.memo (또는 별도 Note)
- 기술 포인트: 인라인 편집, 자동 저장(debounce).

### 4.10 예산 / 정산 대시보드 ⭐
- 설명: 여행 지출을 카테고리별로 기록하고, **1인당 정산(더치페이)**과 차트로 보여준다.
- 주요 동작:
  - 지출 추가: 금액·카테고리(식비/교통/숙박/쇼핑)·결제자·분담 인원
  - 대시보드: 카테고리별 지출 도넛/바 차트, 일자별 누적 라인, "누가 누구에게 얼마" 정산 요약
- 데이터: Expense(id, tripId, amount, currency, category, payerId, sharedWith[], date)
- 기술 포인트: **차트 시각화(recharts) + 집계 로직 + 데이터 테이블**. 경력의 "통계 대시보드·차트" 강점이 가장 직접적으로 드러남.

## 5. 대표 사용자 플로우

1. **여행 만들기**: 로그인 → "새 여행" → 마법사(제목/지역/기간/멤버/권한) → 빈 여행 생성
2. **장소 모으기**: 지도 검색 → 핀 추가 → 폴더("맛집")에 저장 (아직 일정 아님)
3. **일정 짜기**: 저장 장소 중 골라 특정 날짜에 등록 → 드래그로 순서 정리 → 동선 자동 표시
4. **함께 편집**: 친구 초대(편집자) → 동시 접속해 일정 수정, 커서 보임
5. **예산 관리**: 지출 기록 → 대시보드에서 카테고리별·1인당 정산 확인
6. **공유**: 부모님께는 뷰어 링크로 일정만 공유

## 6. 데이터 모델 초안

```
User        : id, name, email, avatar
Trip        : id, title, icon, country, region, startDate, endDate, ownerId
Membership  : tripId, userId, role(owner|editor|viewer)
Folder      : id, tripId, name
Place       : id, tripId, folderId, name, lat, lng, category, memo
ItineraryItem: id, tripId, date, order, placeId, startTime?, note?
Expense     : id, tripId, amount, currency, category, payerId, sharedWith[], date
```

관계: Trip 1—N (Folder, Place, ItineraryItem, Expense, Membership). Place는 저장만 된 상태일 수 있고, ItineraryItem이 특정 날짜·순서로 Place를 참조.

## 8. 기술 스택 (목표 스택 매핑)

| 영역 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | **Next.js 15** (App Router) | 학습/적용 목표 |
| 언어 | **TypeScript** | |
| 상태관리(클라) | **Zustand** | 최근 도입 스택 어필 |
