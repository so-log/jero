# 2차 구현 설계 — 미구현 기능 (A~F)

> 미구현·부분구현 6종의 **구현 설계**를 정리한다. 각 항목: 근거 / 현재 상태 / 설계(데이터·RLS·RPC·훅·UI 배선) / 검증 / 규모.
> 기존 규약(seam 패턴, §7.2 단일 출처, §7.1 컴포넌트 직접 fetch 금지, §8 보안)을 따른다. 갭 요약은 [`../remaining-features.md`](../remaining-features.md), 계약은 [`데이터모델_계약.md`](./데이터모델_계약.md).
> 작성일: 2026-07-08 · 상태: 설계 초안(GATE 2 검토 대상)

**권장 구현 순서**: D(OAuth·설정) → A(커서) → B(폴더) → C(템플릿) → E(통계) → F(메모). 각 항목은 독립적이라 순서 조정 가능.

## A. 실시간 커서

- **근거**: 기능명세서 §4.8, 기획 04 §11(마지막 항목).
- **현재**: `components/map/LiveCursorLayer.tsx` + `useMockCursors`로 **목 데이터**만 렌더. `useTripRealtime`에 "커서 lat/lng는 후속" 명시. presence 아바타(broadcast heartbeat)는 실동작.

### 설계
- **전송 경로**: presence와 **동일한 private 채널**(`trip:<id>`)의 broadcast를 재사용. 새 이벤트 `cursor:move` 추가.
- **페이로드**: `{ userId, lat, lng, ts }` (지도 좌표계로 통일 — 화면 px 아님).
- **송신**: 지도 `mousemove`를 **throttle(예: 50~80ms)** 해서 broadcast. 뷰포트 밖/이동 없음이면 스킵. 이탈 시 `cursor:leave`(best-effort) + TTL.
- **수신**: 피어별 최신 좌표 맵 유지 + TTL(예: 5s)로 만료. 본인 커서는 제외(`self` 필터).
- **렌더**: `useMockCursors` → 실제 커서 훅으로 교체. `LiveCursorLayer` 인터페이스(`cursors: LiveCursor[]`)는 **불변** → 컴포넌트 무변경.
- **멤버 색**: 멤버 식별 색(기존 presence 아바타 색)과 동일 소스 사용.

### 검증
- 2계정 2컨텍스트(e2e): A 커서 이동 → B 화면에 해당 색·이름 커서 표시, 이탈 시 사라짐. throttle로 과도한 메시지 없음(네트워크 확인).
- **규모**: M · 새 마이그레이션 불필요(broadcast 재사용). 새 기획문서 불필요.

## B. 폴더 관리 (생성 / 이름변경 / 삭제 / 이동)

- **근거**: 기능명세서 §4.3, 기획 06(장소).
- **현재**: `usePlacesQuery`가 `folder` 테이블을 조회해 **읽기·필터**는 실동작. `FolderSidebar`의 "폴더 추가" 버튼은 **onClick 미배선(죽은 버튼)**, 폴더 쓰기 훅 없음.

### 설계
- **계약 확인**: `folder`(id, trip_id, name, 정렬/색?) 테이블 + RLS(멤버 CRUD, 0002 포함 여부 확인 — 없으면 정책 추가). place.`folder_id` FK.
- **훅(신규, `features/place/api`)**:
  - `useUpsertFolder(tripId)` — 생성/이름변경(id 유무 분기), 낙관적 `setQueryData(['places',tripId])`, 무효화 `['places',tripId]`.
  - `useDeleteFolder(tripId)` — 삭제 시 소속 장소의 `folder_id` 처리(전체로 이동 = null 또는 "미분류"). 낙관적 + 무효화.
  - 장소 폴더 이동: 기존 `useUpsertPlace`에 `folder_id` 변경 경로 사용(추가 훅 불필요).
- **UI 배선**: `FolderSidebar` "폴더 추가" 버튼 → 인라인 입력/모달로 이름 입력 → `useUpsertFolder`. 폴더 우클릭/더보기 → 이름변경·삭제(editor+). viewer는 관리 UI 비노출.
- **권한**: 멤버(owner/editor)만 폴더 CRUD. 서버 RLS 재검증(§8.2).

### 검증
- 컴포넌트/e2e: 폴더 추가 → 사이드바 반영, 장소를 폴더에 지정 → 개수 갱신, 폴더 삭제 시 장소 미분류 이동. 낙관적 갱신·롤백 테스트.
- **규모**: M · RLS 정책만 필요 시 소규모 마이그레이션. 새 기획문서 불필요(시안 buildFolders 존재).

## C. 템플릿 복제로 시작

- **근거**: 기능명세서 §4.2 Step 4, 기획 03(여행 생성).
- **현재**: 마법사 Step4 UI(빈 여행/템플릿) + `TRIP_TEMPLATES` 있음. `create_trip` RPC에 `-- TODO: startMode='template' 시 템플릿 place 복제`(0002:217) — **실제 복제 미동작**, 시드 데이터 없음.

### 설계
- **템플릿 데이터 구조(신규)**: 템플릿 = { id, title, country/region, 기간(일수), folders[], places[](category·area·lat·lng·memo·Day/순서·time) }. 저장 위치 결정:
  - (a) **시드 테이블**(`trip_template`, `template_place`) — 서버에서 복제(권장, 확장 용이).
  - (b) 코드 상수(`lib/templates`) — 클라에서 생성 요청. 간단하나 유지보수 분산.
  - → **(a) 권장**. GATE 2에서 확정.
- **복제 로직**: `create_trip` RPC 확장 — `start_mode='template'` + `template_id`면 트랜잭션으로 folder·place(일정 배정 포함)를 새 trip에 복사. 날짜는 새 여행 startDate 기준으로 Day 오프셋 매핑.
- **권한/검증**: RPC는 security definer로 소유자 = 호출자 설정. 복제 결과 무효화 `['trips']`·`['places',newId]`.
- **시드 준비**: 대표 템플릿 2~3개(예: 도쿄 3박4일, 제주 2박3일) 데이터 작성.

### 검증
- e2e: 템플릿 선택 → 생성 → 워크스페이스에 폴더·장소·일정이 복제되어 렌더. 빈 여행 경로 회귀.
- **규모**: M~L · **마이그레이션 필요**(시드 테이블/데이터 + RPC). **경미한 설계 보강**(템플릿 스키마)을 계약에 추가.

## D. 구글 소셜 로그인 (설정 위주)

- **근거**: 기능명세서 §4.1, 기획 13(인증 보조).
- **현재**: `useAuth.googleLogin`(`signInWithOAuth({ provider:'google' })`) + `AuthPanel` 버튼 + `/auth/callback` 라우트 존재. **Supabase에 Google provider 미설정**이라 미동작.

### 설계 (코드 변경 최소, 설정 중심)
- **콘솔 설정(사용자)**:
  1. Google Cloud → OAuth 동의 화면 + OAuth 클라이언트 ID 발급(웹). 승인된 리디렉션 URI에 Supabase 콜백(`https://<project>.supabase.co/auth/v1/callback`) 등록.
  2. Supabase → Authentication → Providers → Google 활성화, Client ID/Secret 입력.
  3. Supabase → URL Configuration에 앱 도메인(`https://jero-travel.vercel.app`, `http://localhost:3000`) 등록.
- **코드 확인(내가)**: `signInWithOAuth`의 `redirectTo`가 `/auth/callback`로 맞는지, 콜백 라우트가 `code`→세션 교환 후 `/trips`로 리다이렉트하는지. `on_auth_user_created` 트리거가 OAuth 가입 시에도 profile 프로비저닝하는지(0001 확인).
- **엣지**: 동일 이메일의 이메일가입 ↔ 구글 계정 연결 정책 확인(중복 계정 방지 or 링크).

### 검증
- 프로덕션/로컬: 구글 버튼 → 동의 → `/trips` 도달, profile 자동 생성, 재로그인 정상. 콜백 4xx/5xx 0.
- **규모**: S(설정 위주) · 마이그레이션·기획문서 불필요.

## E. 여행 통계 (설계)

- **근거**: 기획 15(여행 통계) — 신규.
- **현재**: 미구현.

### 설계
- **집계(순수 함수)**: `computeTripStats(places, trip)` → `{ totalDistanceKm, perDay[{day, km, count}], byCategory[{category, count}], byArea[] }`. 이동거리 = 연속 일정 좌표 간 **Haversine** 합(좌표 없는 장소 제외).
- **데이터**: 기존 `usePlacesQuery` 재사용(신규 쿼리·마이그레이션 불필요).
- **뷰**: 워크스페이스 5번째 세그먼트 `?view=stats`(권장) → `features/stats`(요약 카드 + Recharts 차트). 카테고리 색 = `lib/constants/category.ts` 단일 출처.
- **재사용**: 동일 셀렉터를 팜플렛(14) "여행지 소개"에 임베드.
- **(선택) AI 하이라이트**: 서버 라우트가 집계 결과로 요약 카피 제안(지표는 결정적).

### 검증
- 단위: `computeTripStats` 입출력(거리·분포·빈 케이스). 컴포넌트: 카드·차트 렌더, 빈 상태.
- **규모**: M · 마이그레이션 불필요. 기획문서(15) 선행.

## F. 장소 메모 인라인 자동저장 (마이너)

- **근거**: 기능명세서 §4.9 기술 포인트("인라인 편집, 자동 저장 debounce").
- **현재**: 메모 저장은 오버레이 폼 방식. 인라인 자동저장 미구현.

### 설계
- 장소 상세/카드 메모를 인라인 편집 가능하게 + **debounce(예: 600ms) 자동저장** → 기존 `useUpsertPlace`(memo만 patch) 재사용, 낙관적 갱신. 저장 상태 표시("저장됨").
- **권한**: editor+만 편집. viewer 읽기.

### 검증
- 컴포넌트: 입력 → debounce 후 1회 저장 호출, 실패 롤백. **우선순위 낮음(선택)**.
- **규모**: S · 마이그레이션·기획문서 불필요.
