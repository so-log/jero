# 팜플렛 내보내기 — 구현 설계

> 기획 [`../planning/14_팜플렛_내보내기.md`](../planning/14_팜플렛_내보내기.md)와 프로토타입 시안 `docs/design/prototype/팜플렛 내보내기.dc.html`를 구현으로 옮기기 위한 설계.
> 기존 규약 준수: seam 패턴, §7.1(컴포넌트 직접 fetch 금지), §7.2(단일 출처), §8(보안). 계약은 [`데이터모델_계약.md`](./데이터모델_계약.md).
> 작성일: 2026-07-08 · 상태: 설계 초안(GATE 2 검토 대상)

## 0. 범위 · 확정 전제 (기획 §12)
- 디자인 = **테마 프리셋**(차분 패턴 계열 + 일러스트 씬 계열) + **AI 추천**(테마·커버 문구 제안, 렌더는 결정적).
- 출력 = **PDF 내보내기**.
- QR = **기존 공유 링크(`/share/[token]`) 재사용**.
- 섹션 = 표지 · 일정표 · 준비물 · 여행지 소개 · QR (체크 선택).
- 유료 게이팅은 후속(MVP는 기능만). viewer 비노출.

## 1. 아키텍처 · 파일 구성

라우트: `app/(app)/trips/[id]/pamphlet/page.tsx`(설정 + 미리보기). PDF 렌더 전용 라우트 별도(아래 §5).

```
features/pamphlet/
├─ components/
│  ├─ PamphletExportView.tsx   # 2분할: 설정 패널 + A4 3단 미리보기
│  ├─ SettingPanel.tsx         # 섹션 체크박스 · 테마 칩 · AI 추천 · 준비물 편집 · 내보내기
│  ├─ PamphletPreview.tsx      # faces() → 패널 배치(미리보기/인쇄 공용)
│  └─ panels/ (CoverPanel·DayPanel·PrepPanel·IntroPanel·QrPanel)  # 시안 패널 이관
├─ lib/
│  ├─ faces.ts                 # 활성 섹션 → 앞/뒤 면 배열(시안 faces() 이관)
│  └─ prep.ts                  # 기본 준비물 목록
├─ api/
│  ├─ usePamphletData.ts       # trip·places 조회(기존 쿼리 재사용) → 팜플렛 뷰모델
│  ├─ useThemeSuggest.ts       # AI 테마·커버 추천(서버 라우트 호출)
│  └─ useExportPdf.ts          # PDF 생성 요청(서버 라우트) → 다운로드
└─ store/pamphletStore.ts      # 섹션 선택·테마·준비물(클라 UI 상태, Zustand)
```

- 테마 상수는 도메인 독립이라 `lib/constants/pamphletThemes.ts`로(단일 출처).
- 미리보기(`PamphletPreview`)와 PDF 렌더가 **동일 컴포넌트를 공유**한다(미리보기 = PDF 프리뷰 = 결과물 일치, 기획 비기능 요건).

## 2. 데이터 (재사용 우선)
- **여행·일정**: 기존 `usePlacesQuery`(trip 헤더 + places) + `deriveDays`(공유 계약) 재사용 → Day별 일정. **신규 쿼리·마이그레이션 없음.**
- **공유 토큰(QR용)**: 기존 share 도메인의 `issueShareLink` 재사용. 링크 없으면 발급(읽기 전용 스코프·만료, §8.2). QR은 `/share/[token]` 인코딩.
- **테마**: `pamphletThemes` 상수 `{ key, name, palette{ink,accent,accent2,wash,soft,line,chipBg}, art }`. `art` = 패턴/씬 렌더 키.
- **AI 추천 결과**: `{ themeKey, coverCopy }`(저장 안 함, 제안용).
- **UI 상태(Zustand)**: `{ sections{cover,schedule,prep,intro,qr}, themeKey, prep[] }`. 서버 저장 안 함(기획 §4).

## 3. 테마 시스템 (단일 출처 + 씬 렌더러)
- 프로토타입 `THEMES`(beach·city·cherry·snow·food·basic + 씬 계열)를 `lib/constants/pamphletThemes.ts`로 이관. 색·이름·art 키만 상수, 하드코딩 금지(§7.1).
- **art 렌더러**(`features/pamphlet/lib/art.tsx`, 결정적 SVG):
  - 패턴 계열: `waves·city·petals·flakes·dots·grid`(시안 `pattern()` 이관).
  - **씬 계열(신규, 게임느낌)**: `sun·meadow·forest·sea·train·airplane` — 표지면 배경에 크게 깔리는 플랫 일러스트 씬. 팔레트 색(accent/accent2)만으로 그려 테마 팔레트와 일관. **정적**(PDF 출력 전제, 애니메이션 없음).
  - 씬 시안은 Claude Design 산출물(`docs/design/prototype/팜플렛 내보내기.dc.html`)을 기준으로 SVG를 정리해 이관.
- 표지면 = 씬 배경 강조, 일정면·뒷면 = 같은 팔레트의 스트립·아이콘 색만(기획 §3).

## 4. 면 구성 · 레이아웃 (A4 3단)
- 규격: **A4(210×297mm)**, 3단 접이 = 한 면당 3패널, 앞뒤 = **6패널**. 접혀 들어가는 안쪽 패널은 폭을 약간 좁게(≈2mm) 두어 접힘 시 겹침 방지.
- **면 매핑**(시안 `faces()` 이관, `lib/faces.ts`):
  - 앞면(펼침): 표지 / 일정 Day1 / Day2 (선택 섹션에 따라 가감)
  - 뒷면(펼침): 잔여 일정 / 준비물 / 여행지 소개 / QR
  - 접힘 순서 고려해 표지가 접었을 때 최외곽에 오도록 패널 순서 배치.
- 섹션 토글 시 `faces()`가 면 배열을 재계산 → 미리보기·PDF 동일 반영. 일정이 많으면 Day 패널이 다음 면으로 흐르도록 페이지네이션.
- 패널 컴포넌트는 시안(coverPanel·dayPanel·prepPanel·introPanel·qrPanel)을 React 컴포넌트로 이관, 카테고리 색·아이콘은 `lib/constants/category.ts` 단일 출처.

## 5. PDF 내보내기 (핵심 기술 결정)

> **구현 현황 (2026-07-08)**: 서버 headless Chromium을 시도했으나 **이 환경 Node 20 vs `@sparticuz/chromium`의 node22 요구 충돌 + `--ignore-engines` 강제 시 Vercel 프로덕션 빌드 파손 위험**으로 설치 불가 → **아래 "대안(경량) = `window.print()` 폴백"으로 구현**하고 `@sparticuz`는 되돌려 배포 안전을 유지했다. 인쇄 라우트가 미리보기와 동일 패널/`faces`를 재사용하므로 결과는 일치한다. **서버 PDF 승격은 Node 22 환경 확보 시 후속.**

- **방식(초기 권장, 미채택)**: 서버 라우트 `POST /api/pamphlet/export`에서 **headless Chromium**(`puppeteer-core` + `@sparticuz/chromium`, Vercel 서버리스 호환)로 **인쇄 전용 라우트**를 렌더 → A4 landscape·`printBackground:true`로 PDF 버퍼 생성 → 다운로드 응답.
  - 인쇄 라우트: `app/(app)/trips/[id]/pamphlet/print/page.tsx` — `PamphletPreview`와 **같은 컴포넌트**를 `@page { size: A4 landscape; margin: 0 }` print CSS로 렌더. 쿼리로 `theme`·`sections`·`token` 전달.
  - 장점: 벡터 텍스트·정확한 A4·미리보기와 100% 일치.
- **대안(경량)**: 클라이언트 `window.print()`(같은 print 라우트) — PDF 저장은 브라우저 대화상자 의존. 서버리스 크로미움 도입이 부담이면 1차로 이 방식, 이후 서버 PDF로 승격.
- **결정 필요(GATE 2)**: 서버 headless(품질·무인 다운로드) vs 클라 print(간단). 기획은 "PDF 내보내기" 선택 → **서버 headless 권장**, 인프라 부담 시 print 폴백.

## 6. QR
- `qrcode` 라이브러리로 `/share/[token]` 인코딩(SVG). 토큰 없으면 `issueShareLink`로 발급 후 생성. 색은 테마 `ink`(시안 `qrGrid` 대체 = 실제 QR).
- QR 스코프 = 읽기 전용 공유(민감 필드·예산 제외, §08·§8.2). 만료·재발급 규약 준수.

## 7. AI 테마 · 커버 추천
- 서버 라우트 `POST /api/pamphlet/theme-suggest`: 여행 메타(나라·지역·기간·계절·카테고리 분포)로 `{ themeKey(프리셋 중), coverCopy }` 제안. **지표·렌더는 결정적**, AI는 제안만.
- 세션·멤버십 재검증. 실패 시 기본 테마 유지(§5 상태).

## 8. 권한 · 보안
- **멤버(owner/editor)** 만 팜플렛 화면·내보내기. viewer 비노출(§7.1은 UI, 서버 라우트에서 멤버십 재검증 §8.2).
- `/api/pamphlet/*` 라우트는 세션 + trip 멤버십 확인 후 처리. 인쇄 라우트도 멤버 세션 검증(비멤버 접근 차단).
- QR 대상만 공개(읽기 전용 토큰). 팜플렛 자체는 멤버 전용.

## 9. 훅 · seam 매핑
| seam(훅) | 내부 |
|---|---|
| `usePamphletData(tripId)` | `usePlacesQuery` 재사용 → 뷰모델(days·prepDefault·highlights) |
| `useThemeSuggest(tripId)` | `POST /api/pamphlet/theme-suggest` |
| `useExportPdf(tripId)` | `POST /api/pamphlet/export` → blob 다운로드 |
| `useShareActions`(기존) | QR용 토큰 발급/조회 재사용 |

컴포넌트 직접 fetch 금지(§7.1). 섹션·테마·준비물은 `pamphletStore`(Zustand).

## 10. 검증
- **단위**: `faces()` 섹션→면 매핑, 테마 resolve, 거리/하이라이트 파생, QR 인코딩.
- **컴포넌트**: 섹션 토글 시 미리보기 면 갱신, 테마 변경 시 색·씬 반영, viewer 비노출, 빈 상태(일정 0 → 일정표 비활성).
- **e2e(실 Supabase)**: 팜플렛 진입 → 섹션·테마 선택 → 내보내기(서버 라우트 200 + PDF content-type/크기) → QR이 `/share/[token]` 열림. 비멤버 라우트 접근 차단.
- `yarn run check` / `build` 그린.

## 11. 규모 · 의존 · 마이그레이션
- **규모**: M~L(패널 이관 + PDF 파이프라인 + 씬 렌더러).
- **마이그레이션**: **불필요**(기존 테이블·공유 토큰 재사용).
- **신규 의존성**: `qrcode`(경량). PDF 서버 방식 채택 시 `puppeteer-core` + `@sparticuz/chromium`(서버 전용). 클라 print 방식이면 추가 의존성 없음.
- **선행/연계**: 씬 테마 시안(Claude Design) 확정 → art 렌더러 이관. 여행 통계(E) 셀렉터를 "여행지 소개"에 재사용 가능(선택).

## 12. 구현 순서(제안)
1. 테마 상수 + art 렌더러(패턴 우선, 씬은 시안 확정분) 이관
2. 패널 컴포넌트 + `faces()` + 미리보기(`PamphletPreview`)
3. 설정 패널(섹션·테마·준비물) + `pamphletStore`
4. QR(공유 토큰 재사용)
5. 인쇄 라우트 + PDF 내보내기(서버 headless 또는 print 폴백)
6. AI 테마 추천(선택, 후속 가능)
7. 검증(단위·컴포넌트·e2e) → 커밋·main

## 13. 구현 현황 · 후속 (2026-07-08)
- **완료(1·2단계)**: 테마 12종(패턴6+씬6)·패널·`faces`·미리보기·설정 패널·QR(공유 토큰 재사용)·인쇄 라우트(`window.print()` 폴백, 멤버십 재검증)·워크스페이스/공유 진입 버튼. e2e 21/21, check(125)·build 그린, main 반영.
- **후속(미반영)**:
  1. **서버 PDF 승격** — Node 22 환경 확보 시 `puppeteer-core`+`@sparticuz/chromium`로 무인 자동 다운로드 전환(현재는 브라우저 인쇄 대화상자).
  2. **준비물 편집분 인쇄 반영** — 현재 인쇄 라우트는 쿼리(theme·sections·token) 범위라 **기본 준비물**만 인쇄됨. 편집한 체크리스트를 인쇄에 반영하려면 준비물을 쿼리/공유 상태로 전달하거나 서버 조회 필요. (소규모)
  3. **AI 테마 추천(§7)** — LLM 인프라 미구성으로 후속.
